# 🎬 模块 6：视频生成

## 6.1 模块定位

基于模块 5（分镜生成）确认的分镜数据和静态画面图片，**以镜头（Shot）为单位**，调用 AI 视频生成服务将静态图片转化为动态视频片段。每个镜头对应一段短视频（通常 2~6 秒），系统支持批量生成、单镜头重新生成和参数调整。

**核心设计理念**：视频生成是整个管线中最耗时、最依赖外部 AI 服务的环节。用户在分镜模块已完成了所有创意决策（景别、运镜、构图、Prompt），视频生成模块的核心价值是**批量自动化**——将静态图片按照分镜设计转化为动态视频片段，同时提供单镜头级别的精细控制能力。系统以镜头为最小单位进行生成，支持局部重新生成，不影响已完成的镜头。

**前置依赖**：模块 5（分镜生成）已完成，分镜已确认（`Project.step >= 6`，`Project.stepLabel = "视频生成"`），且至少有部分镜头已生成画面图片（`Shot.imageUrl` 不为空）。

**输出物**：每个镜头对应的短视频文件（`Shot.videoUrl`），供模块 7（视频合成）消费。

## 6.2 核心概念

### 视频生成流程

```
分镜数据（Shot）
  ├── imageUrl（参考图 / 首帧图）
  ├── prompt（画面描述 Prompt）
  ├── videoPrompt（视频运动描述，可选）
  ├── cameraMovement（运镜方式）
  ├── duration（目标时长）
  └── imageType（keyframe / first_last / multi_grid）
        ↓
    AI 视频生成服务（如 Kling / Wan / Hailuo）
        ↓
    短视频片段（MP4, 2~6s）
        ↓
    Shot.videoUrl + Shot.videoStatus = "completed"
```

### 生成模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 图生视频（Image-to-Video） | 以静态图片为首帧，生成动态视频 | 有 `imageUrl` 的镜头（最常用） |
| 首尾帧生成（First-Last Frame） | 指定首帧和尾帧，AI 补全中间动画 | `imageType = "first_last"` 的镜头 |
| 文生视频（Text-to-Video） | 仅凭 Prompt 生成视频 | 无 `imageUrl` 的镜头（兜底方案） |

### 视频状态（Shot.videoStatus）

| 状态 | 说明 | UI 表现 |
|------|------|---------|
| `pending` | 未开始生成 | 灰色占位图，「生成」按钮 |
| `generating` | 生成中 | 旋转加载动画，进度条 |
| `completed` | 生成完成 | 视频缩略图 + 播放按钮 |
| `failed` | 生成失败 | 错误提示 + 重试按钮 |

### 6.3.1 进入页面与状态检查

用户从分镜生成页面点击「确认分镜，进入视频生成」后跳转到 `/project/[id]/video-gen`。


### VideoGenConfig（视频生成配置，存为 JSON，挂在 Project 上）

```typescript
interface VideoGenConfig {
  // AI 服务商配置
  provider: "kling" | "wan" | "hailuo"
  model: string           // 如 "kling-v1.6-pro"

  // 生成参数
  defaultDuration: "match_shot" | "2s" | "3s" | "4s" | "5s" | "6s"
  motionStrength: "low" | "medium" | "high"
  quality: "fast" | "standard" | "high"

  // 高级参数
  negativePrompt: string
  seed: number | null     // null 表示随机
  concurrency: number     // 并发数，1~5

  // 兜底策略
  noImageFallback: "skip" | "text_to_video" | "placeholder"
  failureRetry: 0 | 1 | 3
}
```

## 6.5 API 设计

### `GET /api/videos/generate-clips?projectId=xxx`

获取指定项目所有镜头的视频生成状态。

**查询参数：**

| 参数 | 必填 | 说明 |
|------|------|------|
| projectId | 是 | 项目 ID |
| episodeId | 否 | 按分集筛选 |

**响应：**

```json
[
  {
    "id": "shot_xxx",
    "index": 3,
    "videoUrl": "https://...",
    "videoStatus": "completed",
    "videoDuration": 3.0,
    "imageUrl": "https://...",
    "prompt": "Close-up of Lin Wanqiu..."
  }
]
```

---

### `POST /api/videos/generate-clips`

发起单个或批量视频生成任务。

**请求体：**

```json
{
  "projectId": "clyyy...",
  "shotIds": ["shot_1", "shot_2"],
  "config": {
    "provider": "kling",
    "model": "kling-v1.6-pro",
    "motionStrength": "medium",
    "quality": "standard"
  },
  "mode": "missing_only"
}
```

**参数说明：**

| 参数 | 必填 | 说明 |
|------|------|------|
| projectId | 是 | 项目 ID |
| shotIds | 否 | 指定镜头 ID 列表。为空时按 mode 策略选取 |
| config | 否 | 生成参数，覆盖项目默认配置 |
| mode | 否 | `missing_only`（默认）/ `overwrite_all` |

**流程：**

1. 校验项目步骤（`step >= 6`）
2. 按策略筛选待生成的镜头列表
3. 将镜头状态更新为 `generating`
4. 异步调用 AI 服务商 API（后台任务队列）
5. 立即返回任务概要，前端通过轮询获取进度

**响应：**

```json
{
  "success": true,
  "queued": 45,
  "generating": 3,
  "skipped": 12
}
```

---

### `POST /api/videos/generate-clips/cancel`

取消正在进行的批量生成任务。

**请求体：**

```json
{
  "projectId": "clyyy...",
  "shotIds": ["shot_1"]
}
```

---

### `GET /api/videos/generate-clips/status?projectId=xxx`

轮询获取批量生成的实时进度（前端每 3 秒轮询一次）。

**响应：**

```json
{
  "total": 60,
  "completed": 15,
  "generating": 3,
  "pending": 39,
  "failed": 3,
  "estimatedMinutes": 8
}
```

---

### `PATCH /api/videos/generate-clips/:shotId`

更新单个镜头的视频生成参数（如 videoPrompt）。

**请求体：**

```json
{
  "videoPrompt": "新的视频生成提示词...",
  "videoDuration": 4
}
```

---

### `POST /api/videos/confirm`

确认视频生成，更新项目进度。

**请求体：**

```json
{
  "projectId": "clyyy..."
}
```

**流程：**

1. 校验该项目至少有 1 个镜头 `videoStatus = "completed"`
2. 更新 Project：`step = 7`，`stepLabel = "视频合成"`
3. 返回更新后的 Project

## 6.6 状态管理（Zustand Store）

```typescript
// store/video-gen-store.ts

interface ShotVideoInfo {
```

## 6.9 上下游数据流

- **向上游**：
  - 从 `Shot` 获取 `imageUrl`（参考图）、`prompt`（画面 Prompt）、`cameraMovement`（运镜）、`duration`（目标时长）——核心输入
  - 从 `Storyboard` 获取场景分组信息
  - 从 `Episode` 获取分集标题——用于展示
  - 从 `Project` 获取 `platform`、`aspectRatio`、`resolution`——输出规格

- **向下游**：
  - 模块 7（视频合成）：消费 `Shot.videoUrl` 获取视频片段
  - 若 `Shot.videoUrl` 为空，模块 7 兜底使用 `Shot.imageUrl` 作为静态帧

## 6.10 AI 服务集成参考

以可灵（Kling）图生视频为例，典型的 API 调用流程：

```typescript
// 1. 发起生成任务
const response = await fetch("https://api.klingai.com/v1/videos/image2video", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${KLING_API_KEY}`,
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    model_name: "kling-v1-6",
    image: shot.imageUrl,           // 参考图 URL
    prompt: shot.videoPrompt ?? shot.prompt,
    negative_prompt: config.negativePrompt,
    cfg_scale: 0.5,
    mode: config.quality === "high" ? "pro" : "std",
    duration: parseDuration(shot.duration),  // "3" | "5"（秒）
  })
})
const { task_id } = await response.json()

// 2. 保存 taskId，更新状态为 generating
await prisma.shot.update({
  where: { id: shot.id },
  data: { videoStatus: "generating", videoTaskId: task_id }
})

// 3. 轮询任务状态（每 5 秒一次）
const pollResult = await fetch(`https://api.klingai.com/v1/videos/image2video/${task_id}`)
const { task_status, task_result } = await pollResult.json()

if (task_status === "succeed") {
  await prisma.shot.update({
    where: { id: shot.id },
    data: {
      videoStatus: "completed",
      videoUrl: task_result.videos[0].url,
      videoDuration: task_result.videos[0].duration
    }
  })
}
```

> **注意**：视频生成任务通常需要 30 秒到 3 分钟不等，必须使用异步轮询机制，不能同步等待。

## 6.11 非功能性需求

| 维度 | 要求 |
|------|------|
| **生成速度** | 单镜头生成时间取决于 AI 服务商（通常 30s~3min），系统不做额外延迟 |
| **后台运行** | 批量生成任务在后台运行，不阻塞 UI，用户可切换分集或关闭页面后台继续 |
| **错误恢复** | 单镜头生成失败不影响其他镜头；失败后记录错误信息，支持单独重试 |
| **并发控制** | 默认并发 3 个，避免超出 AI 服务 API 的速率限制（Rate Limit） |
| **状态持久化** | 生成状态存储在数据库，刷新页面后状态不丢失，轮询自动恢复 |
| **进度反馈** | 批量生成进度实时更新（每 3 秒轮询一次），用户可感知整体进度 |
| **视频存储** | 生成的视频文件存储在本地项目目录或对象存储，URL 持久有效 |
| **兜底机制** | 无参考图的镜头使用文生视频模式；生成失败的镜头在合成时使用静态图替代 |
| **版本管理** | 支持为同一镜头生成多个版本，用户可选择使用哪个版本 |
| **响应式** | 页面布局在不同屏幕宽度下自适应（≥1280px 双栏 / <1280px 单栏） |
| **快捷键** | 镜头详情弹窗支持 ← → 切换上一个/下一个镜头 |
