## 1. 背景与目标

CutGo（秒剧）是一套 AI 驱动的短剧工业流水线工具，涉及**文本生成**（剧本/大纲/角色分析）、**图像生成**（分镜画面/角色头像）、**视频生成**（动态帧/视频合成）、**语音合成**（TTS 配音）四大 AI 能力域。

直接硬编码对接每一家 SDK 会导致：
- 代码高度耦合，模型迭代成本极高
- 无法灵活切换/降级/A-B 测试
- 错误处理与重试逻辑分散，难以统一监控

**目标**：采用 **抽象工厂 + 网关路由** 模式，在 `src/lib/ai/` 下建立统一的 AI 调用层，对上层业务屏蔽底层模型差异。

---

## 2. 整体架构

```
业务模块（剧本/分镜/画面/TTS）
        │
        ▼
  AIProviderFactory          ← 工厂：根据配置实例化对应 Provider
        │
   ┌────┴────┐
   │  Router │               ← 路由：按能力域 + 用户配置选择 Provider
   └────┬────┘
        │
  ┌─────┼──────┬──────────┐
  ▼     ▼      ▼          ▼
LLM   Image  Video       TTS
Provider Provider Provider Provider
  │     │      │          │
OpenAI Midjourney Runway  OpenAI TTS
Claude  SDXL   Pika    ElevenLabs
DeepSeek Flux  Kling    Edge TTS
  ...    ...    ...       ...
```

所有 Provider 均实现对应的标准接口，业务层只依赖接口，不依赖具体实现。

## 4. 标准接口定义

> 路径：`src/lib/ai/types.ts`

### 4.1 通用类型

```typescript
/** 通用流式回调 */
type StreamCallback = (chunk: string) => void;

/** 通用任务状态（用于异步轮询） */
type TaskStatus = 'pending' | 'processing' | 'succeeded' | 'failed';

interface AsyncTask {
  taskId: string;
  status: TaskStatus;
  progress?: number;       // 0~100
  resultUrl?: string;
  error?: string;
}
```

### 4.2 文本大模型（LLM）接口

```typescript
interface LLMRequest {
  systemPrompt?: string;
  userPrompt: string;
  temperature?: number;    // 0~2，默认 0.7
  maxTokens?: number;
  stream?: boolean;
}

interface LLMResponse {
  content: string;
  usage?: {
    promptTokens: number;
    completionTokens: number;
    totalTokens: number;
  };
}

interface LLMProvider {
  readonly name: string;
  generateText(input: LLMRequest): Promise<LLMResponse>;
  generateTextStream(input: LLMRequest, onChunk: StreamCallback): Promise<void>;
}
```

### 4.3 图像生成接口

```typescript
interface ImageRequest {
  prompt: string;
  negativePrompt?: string;
  width?: number;
  height?: number;
  style?: string;           // 风格预设标识
  referenceImageUrl?: string;
  seed?: number;
}

interface ImageResponse {
  imageUrl: string;
  seed?: number;
}

interface ImageProvider {
  readonly name: string;
  generateImage(input: ImageRequest): Promise<ImageResponse>;
  /** 部分平台为异步任务，需轮询 */
  generateImageAsync?(input: ImageRequest): Promise<AsyncTask>;
  pollTask?(taskId: string): Promise<AsyncTask>;
}
```

### 4.4 视频生成接口

```typescript
interface VideoRequest {
  prompt: string;
  referenceImageUrl?: string;   // 首帧图
  duration?: number;            // 秒
  aspectRatio?: '9:16' | '16:9' | '1:1';
}

interface VideoProvider {
  readonly name: string;
  /** 视频生成通常为异步，返回任务 ID */
  generateVideo(input: VideoRequest): Promise<AsyncTask>;
  pollTask(taskId: string): Promise<AsyncTask>;
}
```

### 4.5 语音合成（TTS）接口

```typescript
interface TTSRequest {
  text: string;
  voiceId?: string;
  speed?: number;    // 0.5~2.0
  language?: string; // 'zh-CN' | 'en-US'
}

interface TTSResponse {
  audioUrl: string;
  durationMs?: number;
}

interface TTSProvider {
  readonly name: string;
  synthesize(input: TTSRequest): Promise<TTSResponse>;
}
```

## 6. 工厂与路由实现

> 路径：`src/lib/ai/factory.ts`

```typescript
import { getLLMProvider, getImageProvider, getVideoProvider, getTTSProvider } from './providers';

export class AIProviderFactory {
  private static config = () => getAIConfig(); // 从 settings 读取用户配置

  static getLLM(): LLMProvider {
    const { textModel, textApiKey } = this.config();
    return getLLMProvider(textModel, textApiKey);
  }

  static getImage(): ImageProvider {
    const { imageModel, imageApiKey, comfyuiUrl } = this.config();
    return getImageProvider(imageModel, imageApiKey, comfyuiUrl);
  }

  static getVideo(): VideoProvider {
    const { videoModel, videoApiKey } = this.config();
    return getVideoProvider(videoModel, videoApiKey);
  }

  static getTTS(): TTSProvider {
    const { ttsModel, ttsApiKey } = this.config();
    return getTTSProvider(ttsModel, ttsApiKey);
  }
}
```

> 路径：`src/lib/ai/providers/index.ts`

```typescript
export function getLLMProvider(model: string, apiKey: string): LLMProvider {
  switch (model) {
    case 'gpt-4o':
    case 'gpt-4-turbo':
      return new OpenAIProvider(apiKey, model);
    case 'claude-3-5-sonnet':
      return new ClaudeProvider(apiKey);
    case 'deepseek-v3':
    case 'deepseek-r1':
      return new DeepSeekProvider(apiKey, model);
    case 'qwen-max':
      return new QwenProvider(apiKey);
    default:
      throw new Error(`未知的文本模型: ${model}`);
  }
}
```

## 7. 各模块调用方式

### 7.1 剧本生成（文本流式）

```typescript
// src/app/api/script/generate/route.ts
import { AIProviderFactory } from '@/lib/ai/factory';

export async function POST(req: Request) {
  const { sceneData } = await req.json();
  const llm = AIProviderFactory.getLLM();

  const stream = new TransformStream();
  const writer = stream.writable.getWriter();

  llm.generateTextStream(
    { systemPrompt: SCRIPT_SYSTEM_PROMPT, userPrompt: buildScriptPrompt(sceneData) },
    async (chunk) => {
      await writer.write(new TextEncoder().encode(`data: ${chunk}\n\n`));
    }
  ).then(() => writer.close());

  return new Response(stream.readable, {
    headers: { 'Content-Type': 'text/event-stream' },
  });
}
```

### 7.2 分镜画面生成（异步任务轮询）

```typescript
// src/app/api/shot/generate-image/route.ts
import { AIProviderFactory } from '@/lib/ai/factory';
import { prisma } from '@/lib/prisma';

export async function POST(req: Request) {
  const { shotId, prompt, negativePrompt } = await req.json();
  const imageProvider = AIProviderFactory.getImage();

  // 1. 提交任务
  const task = await imageProvider.generateImageAsync!({ prompt, negativePrompt });

  // 2. 写入数据库，记录 taskId 和状态
  await prisma.shot.update({
    where: { id: shotId },
    data: { imageTaskId: task.taskId, imageStatus: 'pending' },
  });

  return Response.json({ taskId: task.taskId });
}

// src/app/api/shot/poll-image/route.ts
export async function GET(req: Request) {
  const { searchParams } = new URL(req.url);
  const taskId = searchParams.get('taskId')!;
  const imageProvider = AIProviderFactory.getImage();

  const result = await imageProvider.pollTask!(taskId);

  if (result.status === 'succeeded') {
    await prisma.shot.update({
      where: { imageTaskId: taskId },
      data: { imageUrl: result.resultUrl, imageStatus: 'succeeded' },
    });
  }

  return Response.json(result);
}
```

### 7.3 视频生成（异步 + SWR 轮询）

// 前端：src/hooks/useVideoTask.ts
import useSWR from 'swr';

export function useVideoTask(taskId: string | null) {
  const { data } = useSWR(
    taskId ? `/api/video/poll?taskId=${taskId}` : null,
    fetcher,

## 8. 异步任务管理

视频生成通常需要 60s+，图像生成部分平台也需要 10~30s，需要在数据库中持久化任务状态。

### 8.1 Prisma Schema 扩展

```prisma
model Shot {
  // ... 现有字段
  imageTaskId    String?
  imageStatus    String?   @default("idle")  // idle | pending | processing | succeeded | failed
  imageUrl       String?
  imageError     String?
  imageGeneratedAt DateTime?
}

model VideoClip {
  id          String   @id @default(cuid())
  shotId      String
  taskId      String?
  status      String   @default("idle")
  videoUrl    String?
  error       String?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}
```

### 8.2 轮询策略

| 任务类型 | 建议轮询间隔 | 超时时间 |
|---|---|---|
| 图像生成（SDXL/Flux） | 3s | 120s |
| 图像生成（Midjourney） | 5s | 300s |
| 视频生成（Runway/Pika/Kling） | 5s | 600s |

超时后将任务状态置为 `failed`，前端展示重试按钮。

## 9. 错误处理与降级

### 9.1 统一错误类型

```typescript
export class AIProviderError extends Error {
  constructor(
    message: string,
    public readonly provider: string,
    public readonly code: 'rate_limit' | 'auth_error' | 'timeout' | 'content_policy' | 'unknown',
    public readonly retryable: boolean = false,
  ) {
    super(message);
  }
}
```

### 9.2 重试机制

```typescript
async function withRetry<T>(
  fn: () => Promise<T>,
  maxRetries = 3,
  delayMs = 1000,
): Promise<T> {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (err) {
      if (err instanceof AIProviderError && !err.retryable) throw err;
      if (i === maxRetries - 1) throw err;
      await new Promise((r) => setTimeout(r, delayMs * 2 ** i));
    }
  }
  throw new Error('unreachable');
}
```

### 9.3 降级策略

| 场景 | 主力模型 | 降级模型 |
|---|---|---|
| 文本生成限流 | GPT-4o | DeepSeek-V3 |
| 图像生成失败 | Flux | DALL-E 3 |
| 国内访问不稳定 | OpenAI | 通义千问 / MiniMax |

## 10. 配置与多模型管理

为了支持用户保存多个同类型的模型配置并方便随时切换，系统引入了 `AIModelConfig` 模型。

### 10.1 数据库模型扩展 (Prisma)

```prisma
model AIModelConfig {
  id          String   @id @default(cuid())
  name        String   // 配置名称，如 "我的 GPT-4o"
  type        String   // 模型类型: "llm" | "image" | "video" | "tts"
  provider    String   // 提供商: "openai" | "deepseek" | "comfyui" 等
  model       String   // 模型标识: "gpt-4o" | "flux-1" 等
  apiKey      String   @default("")
  baseUrl     String   @default("")
  config      String   @default("{}") // JSON 字符串，存储额外参数（如 ComfyUI URL）
  
  isDefault   Boolean  @default(false) // 是否为该类型的默认配置
  
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  @@index([type])
}

model Settings {
  id String @id @default("global")

  // 当前选中的各类型模型配置 ID
  activeLLMConfigId   String?
  activeImageConfigId String?
  activeVideoConfigId String?
  activeTTSConfigId   String?

  // ... 其他全局设置
  updatedAt DateTime @updatedAt
}
```

### 10.2 配置读取逻辑

```typescript
// src/lib/ai/config.ts

export async function getActiveConfig(type: 'llm' | 'image' | 'video' | 'tts') {
  const settings = await prisma.settings.findFirst();
  const configId = settings?.[`active${type.toUpperCase()}ConfigId` as keyof Settings];

  if (configId) {
    return await prisma.aIModelConfig.findUnique({ where: { id: configId as string } });
  }

  // 兜底：返回该类型的默认配置
  return await prisma.aIModelConfig.findFirst({ 
    where: { type, isDefault: true } 
  });
}
```

### 10.3 切换与管理功能

1. **配置管理**：用户可以在设置页面添加、编辑、删除多个模型配置。
2. **快速切换**：在设置页或具体的业务操作界面（如剧本生成、分镜生成），提供下拉菜单展示已保存的同类型模型配置，用户切换后更新 `Settings` 中的 `activeXXXConfigId`。
3. **一键测试**：每个配置提供“测试连接”功能，验证 API Key 和 Base URL 是否有效。

## 12. 实现路径建议

| 阶段 | 目标 | 优先实现 |
|---|---|---|
| **Phase 1** | 跑通核心链路 | DeepSeek（文本）+ DALL-E 3（图像）+ Edge TTS（语音） |
| **Phase 2** | 提升质量 | 接入 Flux / SDXL（图像）+ Runway（视频）|
| **Phase 3** | 灵活切换 | 完整 Provider 矩阵 + 用户可在设置页自由选择模型 |

**文本端**：优先流式输出，提升剧本生成的响应体验。  
**图像端**：SQLite + Prisma 记录任务状态，前端 SWR 轮询展示进度。  
**视频端**：异步任务轮询模式，数据库持久化 `task_status`，前端 React Query / SWR 定时轮询，60s+ 超时兜底。
