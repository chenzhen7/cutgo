# 🎬 模块 6：视频合成

## 6.1 模块定位

基于模块 5（分镜生成）确认的分镜数据和画面图片，**以分集（Episode）为单位**，将静态图片序列合成为完整的短剧视频。系统自动为每个镜头叠加字幕、配音（TTS）、BGM 和转场效果，最终输出符合目标平台规格的 MP4 文件。

**核心设计理念**：视频合成是整个管线的"出片"环节。用户在分镜模块已完成了所有创意决策，视频合成模块的核心价值是**自动化**——一键完成字幕、配音、BGM、转场的叠加合成，同时提供精细调整的能力。系统以分集为单位进行合成，每集独立输出一个 MP4 文件，用户可以按集预览、调整和导出。

**前置依赖**：模块 5（分镜生成）已完成，分镜已确认（`Project.step >= 6`，`Project.stepLabel = "视频合成"`），且至少有部分镜头已生成画面图片（`Shot.imageUrl` 不为空）。

**输出物**：按集输出的 MP4 视频文件（本地存储），供模块 7（导出发布）消费。

## 6.2 核心概念

### 视频合成流程

```
分镜数据（Shot[]）
  ├── imageUrl（画面图片）
  ├── dialogueText（台词/字幕文本）
  ├── duration（镜头时长）
  └── actionNote（动作备注）
        ↓
    FFmpeg 合成管线
        ↓
  ├── 图片序列 → 视频轨道
  ├── TTS 生成 → 配音轨道
  ├── BGM 文件 → 背景音乐轨道
  ├── 字幕文件（SRT/ASS）→ 字幕轨道
  └── 转场效果 → 镜头间过渡
        ↓
    输出 MP4（1080×1920, 9:16）
```

### 合成模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 快速合成 | 使用默认参数一键合成，无需配置 | 快速预览效果 |
| 精细合成 | 逐集调整字幕样式、配音、BGM、转场 | 正式输出 |

### 字幕类型

| 类型 | 说明 | 来源 |
|------|------|------|
| 对白字幕 | 角色台词，显示在画面下方 | `Shot.dialogueText`（类型为对白的剧本行） |
| 旁白字幕 | 叙述性文字，显示在画面上方或居中 | `Shot.dialogueText`（类型为旁白的剧本行） |
| 无字幕 | 纯画面镜头 | `Shot.dialogueText` 为空 |

### 转场效果

| 效果 | 英文 | 说明 | 适用情绪 |
|------|------|------|---------|
| 淡入淡出 | Fade | 画面渐隐渐现 | 时间流逝、场景切换 |
| 硬切 | Cut | 直接切换，无过渡 | 快节奏、紧张对话 |
| 溶解 | Dissolve | 两帧叠化过渡 | 梦境、回忆、温情 |
| 滑动 | Slide | 画面向一侧滑入 | 现代感、动感 |
| 缩放 | Zoom | 放大/缩小过渡 | 强调、冲击感 |
| 无转场 | None | 同淡入淡出但时长为 0 | 同场景连续镜头 |

### 配音（TTS）

系统支持调用 TTS（文字转语音）服务，将镜头的台词文本转换为语音：

| 参数 | 说明 |
|------|------|
| 语音角色 | 按角色分配不同的 TTS 声线（男声/女声/儿童声等） |
| 语速 | 0.5x ~ 2.0x，默认 1.0x |
| 语调 | 平静 / 激动 / 低沉 / 欢快 |
| 旁白声线 | 统一的旁白配音声线 |

### 6.3.4 合成任务管理

#### A. 发起合成

用户配置完成后，点击「开始合成」按钮：

1. 系统校验前置条件（画面图片、分镜数据）
2. 弹出合成确认弹窗，展示合成摘要：
   - 待合成分集数量
   - 总镜头数 / 已有画面镜头数
   - 预估合成时间
   - 输出文件路径
3. 用户确认后，创建合成任务并开始执行

#### C. 合成步骤

每集视频的合成分为以下步骤：

| 步骤 | 说明 | 耗时估算 |
|------|------|---------|
| 1. 准备素材 | 下载/验证画面图片，检查缺失图片 | 5-10s |
| 2. 生成配音 | 调用 TTS API 生成各镜头语音文件 | 10-30s（取决于台词量） |
| 3. 生成字幕文件 | 根据镜头时长生成 SRT/ASS 字幕文件 | < 1s |
| 4. FFmpeg 合成 | 图片序列 + 配音 + BGM + 字幕 + 转场 → MP4 | 15-60s（取决于分集时长） |
| 5. 后处理 | 视频质量检查、文件大小统计 | 2-5s |

#### D. 合成状态

| 状态 | 说明 | UI 表现 |
|------|------|---------|
| `idle` | 未开始合成 | 显示配置面板 + 开始按钮 |
| `preparing` | 准备素材中 | 进度条 + 步骤说明 |
| `tts_generating` | 生成配音中 | 进度条 + 当前处理的镜头 |
| `subtitle_generating` | 生成字幕中 | 进度条 |
| `compositing` | FFmpeg 合成中 | 进度条 + 实时帧数 |
| `completed` | 合成完成 | 预览按钮 + 文件信息 |
| `error` | 合成失败 | 错误信息 + 重试按钮 |
| `paused` | 已暂停 | 继续按钮 |

interface VideoCompositionConfig {
  // 字幕配置
  subtitle: {
    enabled: boolean

    narrationPosition: "top_center" | "bottom_center" | "middle_center"

    fontSize: number
    fontColor: string
    strokeColor: string
    strokeWidth: number
    backgroundColor: string
    backgroundOpacity: number
    borderRadius: number
  }

    enabled: boolean
    speed: number           // 0.5 ~ 2.0
    narrationVoice: string  // TTS 声线 ID
    characterVoices: {      // 角色声线映射
      [characterName: string]: string  // 声线 ID
    }
  }

  // BGM 配置
  bgm: {
    enabled: boolean
    masterVolume: number    // 0 ~ 100
    fadeInDuration: number  // 秒
    fadeOutDuration: number // 秒

      fileUrl: string       // BGM 文件路径
      startTime: number     // 在视频中的开始时间（秒）
      endTime: number       // 在视频中的结束时间（秒）
      volume: number        // 0 ~ 100
      loop: boolean
    }[]
  }

  // 转场配置
  transition: {
    defaultEffect: "fade" | "cut" | "dissolve" | "slide" | "zoom" | "none"
    defaultDuration: number  // 秒
    sceneTransitionDuration: number  // 场景切换时的转场时长
    episodeStartFade: boolean
    episodeEndFade: boolean

        inDuration: number
        outEffect: string
        outDuration: number
      }
    }
  }

  // 输出配置
  output: {
    resolution: "1080x1920" | "720x1280" | "540x960"
    fps: 24 | 30 | 60
    videoBitrate: string    // 如 "8M"
    audioBitrate: string    // 如 "192k"
    format: "mp4"

  }
}

### `POST /api/videos`

创建并启动视频合成任务。

**请求体：**

```json
{
  "projectId": "clyyy...",
  "episodeIds": ["ep_1", "ep_2"],
  "config": {
    "subtitle": { "enabled": true, "dialoguePosition": "bottom_center", "..." },
    "tts": { "enabled": false },
    "bgm": { "enabled": true, "masterVolume": 40, "tracks": [...] },
    "transition": { "defaultEffect": "fade", "defaultDuration": 0.5 },
    "output": { "resolution": "1080x1920", "fps": 30, "videoBitrate": "8M", "audioBitrate": "192k", "format": "mp4", "outputDir": "./output/project_name" }
  }
}
```

**参数说明：**

| 参数 | 必填 | 说明 |
|------|------|------|
| projectId | 是 | 项目 ID |
| episodeIds | 否 | 指定分集 ID 列表。为空时合成所有分集 |
| config | 是 | 合成配置对象 |

**流程：**

1. 校验项目步骤（`step >= 6`）
2. 读取指定分集的所有 Storyboard 和 Shot 数据
3. 检查画面图片可用性，统计缺失图片
4. 创建 VideoComposition 记录（status: "preparing"）
5. 异步启动合成任务（后台进程）
6. 立即返回任务 ID，前端通过轮询获取进度

**响应：**

```json
{
  "taskId": "clvc...",
  "status": "preparing",
  "episodeCount": 2,
  "totalShots": 48,
  "missingImages": 3
}
```

## 6.9 上下游数据流

- **向上游**：
  - 从 `Shot` 获取 `imageUrl`（画面图片）、`dialogueText`（字幕文本）、`duration`（镜头时长）、`actionNote`（动作备注）——核心输入
  - 从 `Storyboard` 获取场景分组信息，用于转场逻辑（场景切换时使用更长转场）
  - 从 `ScriptScene` 获取 `bgm`（BGM 建议）、`emotion`（情绪基调）——辅助 BGM 和转场选择
  - 从 `ScriptLine` 获取 `type`（对白/旁白/动作）——决定字幕类型和位置
  - 从 `AssetCharacter` 获取角色名称列表——用于 TTS 声线分配
  - 从 `Project` 获取 `platform`、`aspectRatio`、`resolution`——输出规格

- **向下游**：
  - 模块 7（导出发布）：消费 `VideoComposition.outputPath` 获取 MP4 文件路径
  - 消费 `VideoComposition.fileSize` 和 `videoDuration` 用于导出信息展示

## 6.10 FFmpeg 合成命令参考

以下为单集视频合成的 FFmpeg 命令示意（实际实现需根据配置动态生成）：

```bash
# 步骤 1：将图片序列合成为无声视频（每张图片按 Shot.duration 显示）
# 使用 concat demuxer，通过 concat.txt 文件指定每帧时长

# concat.txt 示例：
# file 'shot_001.jpg'
# duration 4
# file 'shot_002.jpg'
# duration 3
# ...

ffmpeg -f concat -safe 0 -i concat.txt \
  -vf "scale=1080:1920:force_original_aspect_ratio=decrease,pad=1080:1920:(ow-iw)/2:(oh-ih)/2" \
  -r 30 -c:v libx264 -preset fast -crf 18 \
  video_only.mp4

# 步骤 2：合并配音轨道（多个 TTS 音频文件按时间轴拼接）
ffmpeg -i tts_001.mp3 -i tts_002.mp3 ... \
  -filter_complex "[0:a][1:a]concat=n=N:v=0:a=1[out]" \
  -map "[out]" tts_merged.mp3

# 步骤 3：最终合成（视频 + 配音 + BGM + 字幕 + 转场）
ffmpeg -i video_only.mp4 \
  -i tts_merged.mp3 \
  -i bgm.mp3 \
  -filter_complex "
    [1:a]volume=1.0[tts];
    [2:a]volume=0.4,afade=t=in:st=0:d=2,afade=t=out:st=55:d=3[bgm];
    [tts][bgm]amix=inputs=2:duration=first[audio]
  " \
  -vf "subtitles=subtitle.ass" \
  -map 0:v -map "[audio]" \
  -c:v libx264 -preset fast -crf 18 -b:v 8M \
  -c:a aac -b:a 192k \
  -movflags +faststart \
  output_ep01.mp4
```

> **注意**：转场效果通过 FFmpeg 的 `xfade` 滤镜实现，需在图片序列合成阶段处理。字幕通过 ASS 格式嵌入，支持丰富的样式控制。

## 6.11 非功能性需求

| 维度 | 要求 |
|------|------|
| **合成速度** | 单集（60s 视频）合成时间控制在 3 分钟内（本地 CPU 合成） |
| **后台运行** | 合成任务在后台运行，不阻塞 UI，用户可继续操作其他功能 |
| **断点续传** | 合成中途暂停后，可从当前步骤继续，不重复已完成的步骤 |
| **错误恢复** | 单集合成失败不影响其他分集；失败后提供详细错误信息和重试 |
| **磁盘空间** | 合成前检查可用磁盘空间，不足时提前警告（预估每分钟视频约 50MB） |
| **画质** | 输出视频清晰度不低于输入图片（不过度压缩），默认 CRF 18 |
| **音视频同步** | 字幕、配音与画面严格同步，误差 < 100ms |
| **TTS 质量** | TTS 生成的语音自然流畅，语速与字幕时长匹配 |
| **进度反馈** | 合成进度实时更新（每 2 秒轮询一次），用户可感知当前步骤 |
| **文件管理** | 输出文件统一存储在项目目录下，文件名包含集序号和版本号 |
| **版本管理** | 保留最近 3 个版本的合成结果，旧版本可手动删除 |
| **预览流畅度** | 内嵌播放器支持 Range 请求，实现流式播放，无需等待完整加载 |
| **响应式** | 页面布局在不同屏幕宽度下自适应（≥1280px 双栏 / <1280px 单栏） |
| **快捷键** | 播放器支持空格键播放/暂停、← → 跳转、↑ ↓ 调节音量 |
