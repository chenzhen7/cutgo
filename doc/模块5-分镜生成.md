# 🎬 模块 5：分镜生成

## 5.1 模块定位

基于模块 4（剧本生成）的结构化剧本数据，**以剧本场景（ScriptScene）为单位**，AI 自动为每个场景生成详细的分镜设计。每个分镜镜头包含镜头类型、镜头运动、画面构图、角色动态、画面描述 Prompt 和时长等信息，为后续画面生成提供精确的视觉蓝图。

**核心设计理念**：一个剧本场景（ScriptScene）可以拆分为 1~N 个分镜（Shot）。剧本是"说什么"，分镜是"怎么拍"。AI 将剧本中的台词、旁白、动作描述转化为具体的镜头语言——包括景别、角度、运镜、构图和画面 Prompt。用户可以按需选择「生成全部分镜」或「指定分集/场景」进行生成，已生成的分镜可以跳过或重新生成。分镜必须"可编辑"——AI 只是辅助，用户拥有最终决定权。

**前置依赖**：模块 4（剧本生成）已完成，剧本已确认（`Project.step >= 5`，`Project.stepLabel = "分镜生成"`）。

**输出物**：结构化的分镜数据（Storyboard + Shot），供模块 6（画面生成）消费。

## 5.2 核心概念

### 镜头类型（Shot Size）


| 类型  | 英文                | 说明             | 适用场景         |
| --- | ----------------- | -------------- | ------------ |
| 远景  | Extreme Wide Shot | 展示整体环境，人物渺小    | 场景交代、空间感、史诗感 |
| 全景  | Wide Shot         | 展示全身人物与环境关系    | 场景建立、人物出场    |
| 中景  | Medium Shot       | 膝盖以上，人物与环境各半   | 对话、日常交互      |
| 中近景 | Medium Close-Up   | 胸部以上           | 对话、情绪传递      |
| 近景  | Close-Up          | 面部或局部特征        | 情绪特写、反应      |
| 特写  | Extreme Close-Up  | 极致局部（眼睛、嘴唇、手部） | 关键道具、情绪高潮    |


### 镜头运动（Camera Movement）


| 运动  | 英文                   | 说明     | 视觉效果        |
| --- | -------------------- | ------ | ----------- |
| 静止  | Static               | 固定机位   | 稳定、客观       |
| 推进  | Push In / Dolly In   | 镜头向前移动 | 聚焦、紧张感递增    |
| 拉远  | Pull Out / Dolly Out | 镜头向后移动 | 揭示环境、疏离感    |
| 横摇  | Pan                  | 水平旋转   | 扫视环境、跟随移动   |
| 竖摇  | Tilt                 | 垂直旋转   | 从上到下/从下到上展示 |
| 跟拍  | Tracking             | 跟随主体移动 | 沉浸感、运动感     |
| 环绕  | Orbit                | 围绕主体旋转 | 戏剧性、强调主体    |
| 升降  | Crane / Jib          | 垂直升降   | 宏大感、视角切换    |
| 手持  | Handheld             | 模拟手持晃动 | 纪实感、紧张不安    |


### 镜头角度（Camera Angle）


| 角度  | 英文          | 说明      | 心理暗示    |
| --- | ----------- | ------- | ------- |
| 平视  | Eye Level   | 与人物视线齐平 | 平等、客观   |
| 俯拍  | High Angle  | 从上往下拍   | 弱小、被压制  |
| 仰拍  | Low Angle   | 从下往上拍   | 强大、威严   |
| 鸟瞰  | Bird's Eye  | 正上方俯视   | 全局、宿命感  |
| 荷兰角 | Dutch Angle | 倾斜画面    | 不安、心理扭曲 |

## 5.3 核心功能拆解

### 5.3.1 分集选择与分镜生成

用户进入分镜生成页面后，系统自动加载已确认的剧本列表。用户可以选择生成方式：

#### A. 生成全部分镜

- 点击「AI 生成全部分镜」按钮
- 系统遍历所有剧本（Script）的所有场景（ScriptScene），逐场景调用 AI 生成分镜
- 如果某些场景已有分镜数据，弹窗提示用户选择：
  - 「跳过已生成」：仅生成尚无分镜的场景
  - 「全部重新生成」：清空所有旧分镜数据，全部重新生成
  - 「取消」

#### B. 选择指定分集生成

- 页面展示分集选择器（Checkbox 列表）
- 用户勾选需要生成分镜的分集
- 点击「生成选中分镜」按钮
- 仅对勾选分集下的所有场景调用 AI 生成
- 如果选中的分集已有分镜，弹窗提示「重新生成将覆盖该分集的现有分镜」

#### C. 单场景生成

- 在分镜编辑器中，每个场景区块提供独立的「重新生成分镜」按钮
- 点击后仅重新生成该场景的分镜
- 适合用户对单个场景的分镜不满意时重新生成

#### 输入数据


| 来源                         | 数据      | 用途                          |
| -------------------------- | ------- | --------------------------- |
| ScriptScene.title          | 场景标题    | 分镜标题参考                      |
| ScriptScene.description    | 场景描述    | 整体画面构思依据                    |
| ScriptScene.location       | 场景地点    | 环境和背景设定                     |
| ScriptScene.emotion        | 情绪基调    | 影响镜头语言选择（紧张→推进/手持，温馨→静止/缓慢） |
| ScriptScene.bgm            | BGM 建议  | 影响节奏感                       |
| ScriptScene.duration       | 场景时长    | 控制镜头数量和节奏                   |
| ScriptScene.characters     | 出场角色    | 角色构图和动态                     |
| ScriptScene.props          | 涉及道具    | 画面元素                        |
| ScriptLine[].type          | 行类型     | 决定镜头设计（对白→中近景，动作→全景/中景）     |
| ScriptLine[].content       | 行内容     | 画面描述 Prompt 的核心素材           |
| ScriptLine[].character     | 说话角色    | 决定镜头主体                      |
| ScriptLine[].emotion       | 角色情绪    | 影响特写和表情描述                   |
| ScriptLine[].parenthetical | 括号注释    | 动作细节                        |
| AssetCharacter[]           | 角色资产    | 角色外貌描述，用于画面 Prompt          |
| AssetScene[]               | 场景资产    | 场景环境描述，用于画面 Prompt          |
| Project.platform           | 目标平台    | 影响画幅和构图（竖屏 9:16）            |
| Project.aspectRatio        | 画幅比例    | 影响构图方式                      |
| Project.stylePreset        | 视觉风格    | 影响整体画面调性                    |
| Project.globalNegPrompt    | 全局负面提示词 | 排除不需要的元素                    |


#### 生成流程

```
用户选择分集/场景 → 逐场景构建 AI Prompt → 调用大模型 → 解析 JSON → 批量写入数据库 → 前端渲染
```

> 同一分集内的场景采用**串行**方式逐场景生成（保证镜头衔接连贯），不同分集间可**并行**生成。每完成一个场景即更新 UI，用户可实时看到进度。

#### AI Prompt 设计要点

- **角色**：你是一位资深分镜师/电影摄影师，擅长将剧本转化为视觉化的分镜设计
- **核心要求**：
  - 基于场景中的每条剧本行（ScriptLine），将其转化为具体的分镜镜头
  - 台词行可以多条合并为一个镜头，也可以单独一个镜头
  - 每个镜头需包含：景别、镜头运动、镜头角度、画面构图描述、画面 Prompt、时长
  - 画面 Prompt 应包含：场景环境、角色外貌/表情/动作、道具位置、光影氛围
  - 镜头设计要考虑情绪递进——开场建立环境，中间推进情绪，高潮特写强化
  - 注意镜头间的衔接和转场逻辑
  - 考虑竖屏（9:16）画幅的构图特点——人物居中、环境简洁、前景虚化
  - 如果有角色资产信息，画面 Prompt 中使用准确的角色外貌描述以保持一致性
- **输出格式**：要求 AI 返回严格的 JSON 结构

#### 生成状态管理


| 状态           | 说明   | UI 表现                           |
| ------------ | ---- | ------------------------------- |
| `idle`       | 未生成  | 显示空状态 + 生成引导                    |
| `generating` | 生成中  | 进度指示器 + 当前正在生成的场景名 + 进度（如 3/20） |
| `completed`  | 生成完成 | 展示分镜内容                          |
| `error`      | 生成失败 | 错误提示 + 重试按钮，已成功的场景保留结果          |

### 5.3.3 分镜编辑器（拖拉拽交互核心）

分镜编辑器是本模块的核心交互组件。采用**横向时间线 + 拖拉拽**的设计范式，让用户以直觉性的方式编排分镜序列——如同在真实的分镜板上挪动卡片。

#### A. 编辑器布局：横向时间线 + 可拖拽镜头卡片

采用**场景分行、镜头横向排列**的时间线布局，每个场景占一行泳道（Swimlane），场景内的镜头卡片从左到右按时间顺序排列，用户可以自由拖拽重排：

```
┌─ 场景 1：机场回归 ── 📍机场出口 | 感慨 | 12s ── [🔄重新生成] [📖剧本] ─┐
│                                                                      │
│  ⠿ ┌─────────┐  ⠿ ┌─────────┐  ⠿ ┌─────────┐                       │
│    │ ┌─────┐ │    │ ┌─────┐ │    │ ┌─────┐ │    ┌ ─ ─ ─ ─ ┐        │
│    │ │     │ │    │ │     │ │    │ │     │ │    │           │        │
│    │ │预览图│ │    │ │预览图│ │    │ │预览图│ │    │  + 添加   │        │
│    │ │     │ │    │ │     │ │    │ │     │ │    │   镜头    │        │
│    │ └─────┘ │    │ └─────┘ │    │ └─────┘ │    └ ─ ─ ─ ─ ┘        │
│    │ 远景 推进 │    │ 近景 静止 │    │ 中景 跟拍 │                        │
│    │   4s    │    │   3s    │    │   5s    │                        │
│    │ "机场大  │    │ "林晚秋  │    │ "她深吸  │                        │
│    │  厅全貌" │    │  面部…" │    │  一口气" │                        │
│    └─────────┘    └─────────┘    └─────────┘                        │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘

┌─ 场景 2：宴会重逢 ── 📍宴会厅 | 震惊 | 20s ── [🔄重新生成] [📖剧本] ──┐
│                                                                      │
│  ⠿ ┌─────────┐  ⠿ ┌─────────┐  ⠿ ┌─────────┐  ⠿ ┌─────────┐      │
│    │ ...     │    │ ...     │    │ ...     │    │ ...     │      │
│    └─────────┘    └─────────┘    └─────────┘    └─────────┘      │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

**布局规则**：
- 每个场景是一条**水平泳道（Swimlane）**，场景间纵向堆叠
- 场景泳道内的镜头卡片从左到右排列，溢出时水平滚动
- 镜头卡片尺寸统一（约 160×220px），顶部为画面预览区域，底部为镜头元信息
- 每个镜头卡片左上角有 `⠿` 拖拽手柄
- 最右侧为虚线框的「+ 添加镜头」占位卡

#### B. 拖拽交互体系

分镜编辑器的拖拽交互基于 `@dnd-kit/core` 实现，支持以下多层级拖拽操作：

##### B1. 镜头卡片拖拽排序（场景内）

- **拖拽手柄**：卡片左上角 `⠿` 图标为拖拽抓取区
- **拖拽范围**：默认限制在同一场景泳道内，水平方向拖拽
- **视觉反馈**：
  - 拖起时卡片半透明化（opacity: 0.5），出现蓝色虚线占位符
  - 拖拽中其他卡片自动让位，平滑动画过渡
  - 放置时卡片弹入目标位置，伴随微弱弹性动画
- **数据更新**：拖拽完成后立即调用 `PUT /api/storyboards/:id/shots/reorder` 持久化新顺序

##### B2. 镜头卡片跨场景拖拽

- **触发方式**：长按拖拽手柄 > 300ms 后解锁跨场景拖拽模式
- **视觉提示**：解锁时卡片边框变为蓝色虚线，提示"可跨场景移动"
- **拖拽范围**：可将镜头从一个场景泳道拖入另一个场景泳道的任意位置
- **数据更新**：
  1. 从原 Storyboard 中移除该 Shot
  2. 加入目标 Storyboard，设定新 index
  3. 两个 Storyboard 的 Shot index 均重新计算
- **API**：新增 `POST /api/storyboards/shots/move`

##### B3. 场景泳道拖拽排序

- **拖拽手柄**：场景泳道左侧的 `≡` 图标
- **拖拽方向**：纵向，调整场景在本集中的展示顺序
- **视觉反馈**：拖起时整条泳道半透明，目标位置出现蓝色横线指示器
- **数据关联**：场景排序实际映射到 ScriptScene 的 index 重排（同步更新剧本模块数据）

##### B4. 镜头时长拖拽调节

- **触发方式**：鼠标悬停在镜头卡片右边缘时，出现一条垂直的拖拽调节手柄
- **拖拽方向**：水平拖拽，向右拉长 = 增加时长，向左缩短 = 减少时长
- **视觉反馈**：
  - 拖拽时卡片宽度实时伸缩，按时长比例映射（1s ≈ 40px）
  - 卡片上方浮现时长气泡实时显示当前值（如 "3s → 5s"）
  - 场景泳道总时长指示器同步更新
- **约束**：最小时长 1s，最大时长 15s，步进 0.5s
- **数据更新**：拖拽结束后 debounce 300ms 调用 `PATCH` 更新时长

#### C. 镜头卡片设计

每个镜头卡片采用紧凑的卡片设计，优化信息密度：

```
┌───────────────────┐
│ ⠿  #1        ✕ 📋│  ← 拖拽手柄 + 序号 + 删除 + 复制
│ ┌───────────────┐ │
│ │               │ │
│ │   画面预览     │ │  ← 预留缩略图区域（160×90px, 9:16 裁切预览）
│ │   (点击放大)   │ │
│ │               │ │
│ └───────────────┘ │
│ ┌─────┐ ┌──────┐ │
│ │ 远景 │ │ 推进 │ │  ← 景别 Badge + 运镜 Badge（可点击切换）
│ └─────┘ └──────┘ │
│ 平视  ·  4s       │  ← 角度 + 时长（时长可直接点击编辑）
│───────────────────│
│ "机场大厅全貌，    │  ← 构图描述（1-2 行截断，点击展开编辑）
│  人流涌动..."     │
│───────────────────│
│ 📝 旁白+动作 ×2   │  ← 关联的剧本行概要
│───────────────────│
│ [✨优化Prompt]     │  ← Prompt 操作按钮
└───────────────────┘
        ├──────┤
        拖拽调节手柄
        (调节时长)
```

**卡片交互**：
- **单击卡片**：打开右侧/底部的**镜头详情编辑面板**（不是弹窗，保持上下文）
- **双击 Badge**：直接弹出下拉切换景别/运镜/角度（快捷修改）
- **点击时长**：进入内联编辑模式，输入新时长
- **右键菜单**：复制、在前方插入、在后方插入、删除、重新生成此镜头

#### D. 镜头详情编辑面板

选中某个镜头卡片后，编辑器右侧滑出**详情面板**（或底部抽屉，取决于屏幕宽度），以表单形式编辑完整属性：

**面板交互**：
- 面板固定在右侧，宽度约 400px，内容可滚动
- 提供「← 上一镜头」「下一镜头 →」快捷切换（键盘 ← → 也可切换）
- 所有修改实时保存（debounce 500ms）
- 关闭面板后回到时间线总览

#### E. 镜头操作

- **拖拽排序**：镜头卡片支持场景内水平拖拽 + 跨场景拖拽
- **拖拽调节时长**：卡片右边缘拖拽手柄调节时长
- **右键菜单操作**：
  - **复制镜头**：复制当前镜头所有属性，在其后方插入副本
  - **在前方插入**：在当前镜头前方插入空白新镜头
  - **在后方插入**：在当前镜头后方插入空白新镜头
  - **删除镜头**：需二次确认
  - **AI 重新生成此镜头**：仅重新生成该镜头的 Prompt
- **添加镜头**：场景泳道最右侧的虚线框「+ 添加镜头」占位卡
- **批量选择**：按住 Shift 点击可多选镜头卡片，支持批量删除、批量修改景别/运镜
- **快捷键**：
  - `Delete` / `Backspace`：删除选中镜头
  - `Ctrl+D`：复制选中镜头
  - `Ctrl+Z` / `Ctrl+Shift+Z`：撤销/重做（V2）

#### F. 场景泳道操作

- **拖拽排序**：场景泳道支持纵向拖拽重排
- **折叠/展开**：点击场景标题栏可折叠泳道（仅显示标题行 + 缩略统计）
- **重新生成分镜**：整个场景重新调用 AI 生成分镜
- **查看原始剧本**：场景标题栏的「📖 剧本」按钮，弹出浮层显示该场景的完整剧本行
- **添加镜头**：泳道内最右侧的占位卡片

### 5.3.4 分镜管理（Storyboard + Shot CRUD）

- 一个 ScriptScene 对应一个 Storyboard（1:1 关系）

#### C. 分镜预览模式

- 每个镜头以缩略图 + 时长条形式排列，条形宽度按时长比例映射

- 预览模式下隐藏所有编辑控件
- 支持点击预览中的镜头条跳转回编辑模式定位到该镜头

### `POST /api/storyboards/generate`

AI 生成分镜（支持全部场景、指定分集或指定场景）。

**请求体：**

```json
{
  "projectId": "clyyy...",
  "episodeIds": ["ep_1", "ep_3"],
  "scriptSceneIds": [],
  "mode": "skip_existing"
}
```

**参数说明：**


| 参数             | 必填  | 说明                                                            |
| -------------- | --- | ------------------------------------------------------------- |
| projectId      | 是   | 项目 ID                                                         |
| episodeIds     | 否   | 指定分集 ID 列表。传入时仅生成这些分集下的场景分镜                                   |
| scriptSceneIds | 否   | 指定场景 ID 列表。传入时仅生成这些场景的分镜（优先级高于 episodeIds）                    |
| mode           | 否   | 冲突处理模式：`"skip_existing"` / `"overwrite"`。默认 `"skip_existing"` |


> 当 `episodeIds` 和 `scriptSceneIds` 均为空时，生成项目下所有场景的分镜。

**流程：**

1. 从数据库读取该项目的 Script 数据（含 ScriptScene + ScriptLine）
2. 读取角色资产（AssetCharacter）和场景资产（AssetScene），用于画面 Prompt 生成
3. 确定待生成的场景列表：
  - 如传入 `scriptSceneIds`，使用指定场景
  - 如传入 `episodeIds`，使用这些分集下的所有场景
  - 否则使用所有场景
4. 根据 `mode` 处理冲突：
  - `skip_existing`：过滤掉已有 Storyboard 的场景
  - `overwrite`：先删除这些场景的旧 Storyboard 数据
5. 读取 Project 配置（platform、aspectRatio、stylePreset、globalNegPrompt）
6. 逐场景调用 AI 大模型生成分镜
7. 解析返回的 JSON，批量创建 Storyboard 和 Shot
8. 返回完整的分镜列表

**响应：**

```json
{
  "storyboards": [...],
  "stats": {
    "storyboardCount": 12,
    "totalShots": 48,
    "totalDuration": "300s",
    "avgShotsPerScene": 4,
    "generatedScenes": 12,
    "skippedScenes": 8
  }
}
```

**错误处理：**

- 项目无已确认的剧本数据 → 返回 400
- 指定的 episodeId / scriptSceneId 不存在 → 返回 400
- AI 服务不可用 → 返回 503
- 部分场景生成失败 → 返回已成功的结果 + 失败场景列表

### 拖拽交互汇总

分镜生成模块的拖拽交互是整个产品中最丰富的，以下汇总所有拖拽场景：

| 拖拽对象 | 拖拽方向 | 触发方式 | 约束条件 | 视觉反馈 | 数据影响 |
|---------|---------|---------|---------|---------|---------|
| 镜头卡片（场景内） | 水平 | 抓取 `⠿` 手柄 | 限制在同一场景泳道内 | 半透明 + 蓝色占位符 + 其他卡片让位动画 | 更新该 Storyboard 内 Shot 的 index |
| 镜头卡片（跨场景） | 自由 | 长按 `⠿` 手柄 >300ms | 可跨越场景泳道边界 | 卡片边框变蓝色虚线 + 目标泳道高亮 | 移动 Shot 到目标 Storyboard，两边 index 重算 |
| 场景泳道 | 纵向 | 抓取 `≡` 手柄 | 限制在同一分集内 | 整条泳道半透明 + 蓝色横线指示器 | 更新 ScriptScene 的 index |
| 镜头时长调节 | 水平 | 拖拽卡片右边缘手柄 | 最小 1s，最大 15s，步进 0.5s | 卡片宽度实时伸缩 + 时长气泡 | 更新 Shot 的 duration |

**技术实现**：基于 `@dnd-kit/core` + `@dnd-kit/sortable` 实现，使用 `SortableContext` 管理同泳道内排序，`DndContext` 管理跨泳道拖拽。时长调节使用原生 `onPointerDown` + `onPointerMove` 实现。

### 交互细节

1. **空状态**
   - 首次进入且无分镜数据时，显示空状态引导：
     - 插图 + "尚未生成分镜"
     - 展示剧本场景列表预览（来自剧本的数据）
     - 「AI 生成全部分镜」主按钮
     - 「选择分集生成」次要按钮
   - 如果剧本未确认（`Project.step < 5`），显示提示并引导返回剧本生成页

2. **AI 生成进度**
   - 生成按钮变为 loading 状态，显示进度："正在生成 第1集·重逢 / 机场回归 (3/20)..."
   - 每完成一个场景，镜头卡片以动画渐入的方式追加到对应的场景泳道中
   - 对应的分集状态实时更新（灰色 → loading → 绿色勾）
   - 支持取消生成（中断后续场景，已完成的保留）

3. **分集导航交互**
   - 左侧分集导航列表，点击切换当前查看/编辑的分集
   - 当前选中的分集高亮显示
   - 每个分集显示：集标题、生成状态、镜头总数
   - 未生成的分集可点击进入后显示该集的场景泳道（空泳道 + 「生成分镜」按钮）

4. **时间线编辑器交互**
   - 中间区域为场景泳道列表，每个泳道内横向排列镜头卡片
   - 泳道内镜头溢出时可水平滚动（鼠标滚轮 + 触控板横滑均支持）
   - 点击镜头卡片选中，右侧详情面板自动展开并显示该镜头的完整属性
   - 按住 Shift 点击可多选镜头（高亮选中态），支持批量删除/修改
   - 所有拖拽操作均有平滑的弹性动画反馈
   - 修改实时保存（debounce 500ms）

5. **镜头详情面板交互**
   - 右侧固定面板，展示选中镜头的完整编辑表单
   - 提供「← 上一镜头」「下一镜头 →」快捷切换（键盘左右箭头也可切换）
   - 景别/运镜/角度使用下拉选择器，带图标辅助识别
   - Prompt 编辑区为多行文本框，提供「AI 优化 Prompt」按钮
   - 优化结果以 diff 形式展示（删除的红色高亮 + 新增的绿色高亮），用户可选择采纳或放弃
   - 关联剧本行区域可手动点击「重新关联」调整
   - 无镜头选中时面板显示空状态提示"点击镜头卡片查看详情"
   - 面板可通过右上角 ✕ 按钮关闭，关闭后中间编辑器区域自动扩展

6. **右键菜单**
   - 右键点击镜头卡片弹出上下文菜单：
     - 复制镜头 (Ctrl+D)
     - 在前方插入空白镜头
     - 在后方插入空白镜头
     - AI 重新生成此镜头
     - 删除 (Delete)
   - 右键点击场景泳道空白区域弹出菜单：
     - 在此处添加镜头
     - 重新生成整个场景分镜
     - 查看原始剧本

7. **确认分镜**
   - 底部固定操作栏
   - 至少有 1 个已生成分镜时按钮可用，否则禁用
   - 点击后弹出确认弹窗，显示摘要：已生成分镜数、总镜头数、总时长
   - 确认后更新项目进度并跳转

## 5.11 AI Prompt 模板参考

```text
你是一位资深分镜师/电影摄影师，擅长将剧本转化为精确的分镜设计。

## 任务
请基于以下剧本场景，为其设计详细的分镜镜头序列。

## 当前场景信息
- 场景标题：{sceneTitle}
- 场景描述：{sceneDescription}
- 场景地点：{sceneLocation}
- 情绪基调：{sceneEmotion}
- BGM 建议：{sceneBgm}
- 目标时长：{sceneDuration}
- 出场角色：{sceneCharacters}
- 涉及道具：{sceneProps}

## 场景剧本行
{scriptLines}

## 角色资产（用于画面 Prompt 中的外貌描述）
{assetCharacters}

## 场景资产（用于画面 Prompt 中的环境描述）
{assetScenes}

## 全局参数
- 目标平台：{platform}
- 画幅比例：{aspectRatio}（竖屏构图）
- 视觉风格：{stylePreset}
- 全局负面提示词：{globalNegPrompt}

## 前一场景最后一个镜头信息
{previousShot}（如有，确保衔接）

## 要求
1. 将剧本行转化为具体的镜头序列，通常一个场景拆分为 2-6 个镜头
2. 合理搭配景别：
   - 场景开头用远景/全景建立环境
   - 对话用中景/中近景
   - 情绪高潮用近景/特写
   - 关键道具/表情用特写
3. 一条台词行可以对应一个镜头，也可以多条合并为一个镜头
4. 每个镜头需包含：
   - shotSize：景别（extreme_wide/wide/medium/medium_close/close/extreme_close）
   - cameraMovement：镜头运动（static/push_in/pull_out/pan/tilt/tracking/orbit/crane/handheld）
   - cameraAngle：镜头角度（eye_level/high/low/birds_eye/dutch）
   - composition：画面构图描述（中文，描述画面中的元素布局）
   - prompt：画面描述（英文，用于 AI 图像生成，包含场景、角色外貌、表情、动作、光影、氛围）
   - negativePrompt：负面提示词（英文，排除不需要的元素）
   - duration：预估时长
   - dialogueText：该镜头期间的台词/旁白文本（如有）
   - actionNote：动作备注（如有）
   - scriptLineIds：关联的剧本行索引列表
5. 画面 Prompt 要求：
   - 英文描述，详细且具体
   - 包含：场景环境（时间、天气、光线）、角色外貌（使用角色资产中的描述）、
     角色表情和动作、道具位置、构图方式、光影效果、整体氛围
   - 末尾标注画幅比例（如 "9:16 vertical frame"）

   - 震惊/冲击：快速推进、特写

   - 激昂/愤怒：跟拍、手持
7. 所有镜头时长之和应接近场景目标时长
8. 如有前一场景的最后一个镜头信息，确保本场景第一个镜头与之自然衔接
```

## 5.12 非功能性需求

| 维度 | 要求 |
|------|------|
| **性能** | 时间线编辑器渲染（含 4 场景泳道 × 4 镜头卡片）应在 300ms 内完成 |
| **拖拽流畅度** | 拖拽过程帧率 ≥ 30fps，let位/放置动画 ≤ 200ms，无明显卡顿或抖动 |
| **AI 响应** | 单场景分镜生成控制在 45s 内，多场景生成时每完成一个即更新 UI |
| **数据安全** | 分镜数据仅存本地 SQLite，不上传到云端（AI 调用除外） |
| **实时保存** | 编辑操作 debounce 500ms 后自动保存，拖拽完成后立即保存 |
| **拖拽体验** | 镜头卡片拖拽有半透明 + 占位符视觉反馈，时长调节有实时宽度映射和气泡提示 |
| **数据一致性** | 删除/排序/跨场景移动操作后，index 自动重新计算 |
| **渐进式生成** | 多场景生成时，已完成的场景结果以动画渐入方式立即可见 |
| **容错性** | 多场景生成中途失败不影响已成功的结果；拖拽中取消（Esc）回到原位 |
| **Prompt 质量** | 生成的画面 Prompt 应详细、具体，直接可用于主流 AI 绘图模型 |
| **镜头衔接** | 相邻镜头的景别、运动应有逻辑衔接，避免跳跃感 |
| **竖屏适配** | 所有 Prompt 和构图建议默认考虑竖屏（9:16）画幅 |
| **触控兼容** | 拖拽交互兼容触控设备（长按触发拖拽、触控横滑滚动泳道） |
| **响应式** | 三栏布局在不同屏幕宽度下自适应（≥1440px 三栏 / 1024-1439px 双栏 / <1024px 单栏） |
| **快捷键** | 支持 Delete 删除、Ctrl+D 复制、← → 切换镜头、Esc 取消拖拽 |
