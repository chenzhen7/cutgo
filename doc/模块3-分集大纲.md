# 📋 模块 3：分集大纲

## 3.1 模块定位

基于模块 2（小说导入）的分析结果，**以章节为单位**，AI 自动将每个章节的内容拆分为一个或多个短剧分集大纲。每集包含集标题、剧情摘要、核心冲突、结尾钩子（cliffhanger）和场景拆分，为后续剧本生成提供结构化蓝图。

**核心设计理念**：章节是大纲生成的基本单元。一个章节可以生成 1~N 集分集大纲，每集归属于明确的来源章节。用户可以按需选择「生成全部章节」或「指定章节」进行生成，已生成的章节可以跳过或重新生成。

**前置依赖**：模块 2（小说导入）已完成，小说已确认导入（`Novel.status = "confirmed"`）。

**输出物**：结构化的分集大纲数据（Episode + EpisodeScene），供模块 4（剧本生成）消费。

---

## 3.2 核心功能拆解

### 3.2.1 章节选择与大纲生成

用户进入分集大纲页面后，系统自动加载已导入的小说章节列表。用户可以选择生成方式：

#### A. 生成全部章节大纲

- 点击「AI 生成全部大纲」按钮
- 系统遍历所有已选中章节（`Chapter.selected = true`），逐章调用 AI 生成分集大纲
- 如果某些章节已有大纲数据，弹窗提示用户选择：
  - 「跳过已生成」：仅生成尚无大纲的章节
  - 「全部重新生成」：清空所有旧数据，全部重新生成
  - 「取消」

#### B. 选择指定章节生成

- 页面左侧/顶部展示章节选择器（Checkbox 列表）
- 用户勾选需要生成大纲的章节
- 点击「生成选中章节大纲」按钮
- 仅对勾选的章节调用 AI 生成
- 如果选中的章节已有大纲，弹窗提示「重新生成将覆盖该章节的现有大纲」

#### C. 单章节生成

- 在章节选择器中，每个章节条目右侧有独立的「生成」按钮
- 点击后仅生成该章节的分集大纲
- 适合用户对单个章节不满意时重新生成

#### 输入数据

| 来源 | 数据 | 用途 |
|------|------|------|
| Chapter.content | 当前章节的原文内容 | 核心素材，AI 基于此拆分分集 |
| Chapter.title | 章节标题 | 提供上下文 |
| Novel.synopsis | 故事大纲 | 提供整体剧情脉络，确保分集连贯 |
| Novel.characters | 角色列表 | 确保每集角色分配合理 |
| Novel.events | 剧情事件 | 提供结构参考 |
| Project.duration | 目标时长（30s/60s/90s） | 控制每集时长和节奏 |
| Project.platform | 目标平台 | 影响叙事风格（如抖音更快节奏） |

#### 生成流程

```
用户选择章节 → 逐章构建 AI Prompt → 调用大模型 → 解析 JSON → 批量写入数据库 → 前端渲染
```

> 多章节生成时采用**串行**方式逐章调用（保证上下文连贯），每完成一章即更新 UI，用户可实时看到进度。

#### AI Prompt 设计要点

- **角色**：你是一位资深短剧编剧，擅长将小说改编为高留存率的竖屏短剧
- **核心要求**：
  - 根据章节内容体量，合理拆分为 1~N 集（短章节可能只需 1 集，长章节可拆为多集）
  - 每集必须有明确的核心冲突（conflict）
  - 每集结尾必须有钩子（cliffhanger），制造悬念让观众追下一集
  - 场景拆分要考虑视觉呈现，每个场景应有画面感
  - 根据目标平台和时长调整节奏
  - 如果是多章节生成，需考虑与前后章节的衔接
- **输出格式**：要求 AI 返回严格的 JSON 结构
- **上下文窗口**：如单章文本过长，分段处理后合并

#### 生成状态管理

| 状态 | 说明 | UI 表现 |
|------|------|---------|
| `idle` | 未生成 | 显示空状态 + 生成引导 |
| `generating` | 生成中 | 进度指示器 + 当前正在生成的章节名 + 进度（如 2/5） |
| `completed` | 生成完成 | 展示分集列表 |
| `error` | 生成失败 | 错误提示 + 重试按钮，已成功的章节保留结果 |

### 3.2.2 分集列表展示与章节筛选

#### A. 分集列表

以卡片列表形式展示所有分集。分集按全局 `index` 排序（跨章节连续编号），每集卡片包含：

| 字段 | 说明 |
|------|------|
| 集序号 | 全局连续编号（1, 2, 3...），跨章节递增 |
| 集标题 | 如"第1集 · 重逢" |
| 来源章节 | 该集所属的章节标题（如"第一章 重生"），以 Badge 形式展示 |
| 剧情摘要 | 本集核心剧情概述（200字内） |
| 核心冲突 | 本集的戏剧冲突 |
| 结尾钩子 | cliffhanger 描述 |
| 预估时长 | 本集总时长（如 60s） |
| 场景数量 | 本集包含的场景数 |
| 涉及角色 | 本集出场的角色列表 |

#### B. 按章节筛选

分集列表上方提供**章节筛选栏**：

- 展示所有章节的筛选标签（Tag/Badge 形式），支持多选
- 默认「全部」选中，显示所有分集
- 点击某个章节标签，仅显示该章节生成的分集
- 支持同时选中多个章节标签
- 筛选标签上显示该章节的分集数量（如"第一章 重生 (3)"）
- 章节标签按章节 index 排序
- 未生成大纲的章节标签显示为灰色/禁用态，标注"未生成"

#### C. 章节生成状态一览

在筛选栏或侧边，每个章节标签旁显示生成状态图标：

| 状态 | 图标/样式 | 说明 |
|------|-----------|------|
| 未生成 | 灰色圆点 | 该章节尚未生成大纲 |
| 已生成 | 绿色勾 | 该章节已有大纲数据 |
| 生成中 | 旋转 loading | 正在生成该章节的大纲 |
| 生成失败 | 红色叹号 | 该章节生成失败 |

### 3.2.3 分集管理（Episode CRUD）

#### A. 添加分集

- 点击「添加一集」按钮
- 弹出编辑弹窗，需选择归属章节（下拉选择）
- 新集 index 自动计算（追加到该章节的最后一集之后）
- 全局 index 自动重新编号

#### B. 编辑分集

- hover 卡片显示编辑按钮
- 点击后弹出编辑弹窗，可修改：集标题、剧情摘要、核心冲突、结尾钩子、预估时长、归属章节
- 修改归属章节时，全局 index 自动重新计算
- 所有修改实时保存到数据库

#### C. 删除分集

- hover 卡片显示删除按钮
- 删除需二次确认
- 删除后全局 index 自动重新编号
- 级联删除该集下所有场景

#### D. 拖拽排序

- 分集卡片支持拖拽调整顺序
- 拖拽完成后自动更新所有集的 index 并持久化

---

### 3.2.4 场景管理（EpisodeScene CRUD）

每集下包含多个场景，场景是分集大纲的最小粒度单元。

#### A. 场景列表展示

在分集卡片内，点击展开可查看场景列表。每个场景包含：

| 字段 | 说明 |
|------|------|
| 场景序号 | 集内编号（1, 2, 3...） |
| 场景标题 | 如"机场回归"、"宴会重逢" |
| 场景摘要 | 场景内容描述 |
| 预估时长 | 该场景时长（如 15s） |
| 出场角色 | 该场景涉及的角色 |
| 情感标签 | 场景情感基调（如"紧张"、"温馨"、"悬疑"） |

#### B. 添加场景

- 在展开的场景列表底部，点击「添加场景」
- 弹出场景编辑弹窗

#### C. 编辑场景

- 点击场景条目进入编辑
- 可修改：标题、摘要、时长、角色、情感标签

#### D. 删除场景

- 场景条目右侧显示删除按钮
- 删除后自动重新编号

#### E. 拖拽排序

- 场景支持集内拖拽排序
- 更新 index 并持久化

## 3.3 数据模型设计

### Episode（分集）

```prisma
model Episode {
  id        String   @id @default(cuid())
  projectId String
  project   Project  @relation(fields: [projectId], references: [id], onDelete: Cascade)

  chapterId String
  chapter   Chapter  @relation(fields: [chapterId], references: [id], onDelete: Cascade)

  index        Int       // 全局连续编号（跨章节）
  title        String
  synopsis     String
  keyConflict  String?
  cliffhanger  String?
  duration     String    @default("60s")

  scenes EpisodeScene[]

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

### EpisodeScene（分集场景）

```prisma
model EpisodeScene {
  id        String   @id @default(cuid())
  episodeId String
  episode   Episode  @relation(fields: [episodeId], references: [id], onDelete: Cascade)

  index      Int
  title      String
  summary    String
  duration   String    @default("15s")
  characters String?   // JSON: ["林晚秋", "陆景琛"]
  emotion    String?

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

### 关联关系更新

在 `Project` 模型中新增：

```prisma
model Project {
  // ... 现有字段
  novel    Novel?
  episodes Episode[]
}
```

在 `Chapter` 模型中新增：

```prisma
model Chapter {
  // ... 现有字段
  episodes Episode[]
}
```

> **核心关系**：
> - Project 1:N Episode（一个项目有多个分集）
> - Chapter 1:N Episode（一个章节可生成多个分集）
> - Episode 1:N EpisodeScene（一个分集有多个场景）

### `POST /api/episodes/generate`

AI 生成分集大纲（支持全部章节或指定章节）。

**请求体：**

```json
{
  "projectId": "clyyy...",
  "chapterIds": ["ch_1", "ch_3"],
  "mode": "skip_existing"
}
```

**参数说明：**

| 参数 | 必填 | 说明 |
|------|------|------|
| projectId | 是 | 项目 ID |
| chapterIds | 否 | 指定章节 ID 列表。为空或不传时，生成所有 `selected = true` 的章节 |
| mode | 否 | 冲突处理模式：`"skip_existing"`（跳过已生成）/ `"overwrite"`（覆盖重新生成）。默认 `"skip_existing"` |

**流程：**

1. 从数据库读取该项目的 Novel 数据（synopsis、characters、events）
2. 确定待生成的章节列表：
   - 如传入 `chapterIds`，使用指定章节
   - 否则使用所有 `selected = true` 的章节
3. 根据 `mode` 处理冲突：
   - `skip_existing`：过滤掉已有 Episode 的章节
   - `overwrite`：先删除这些章节的旧 Episode 数据
4. 读取 Project 配置（platform、duration）
5. 逐章调用 AI 大模型生成分集大纲
6. 解析返回的 JSON，批量创建 Episode 和 EpisodeScene
7. 重新计算所有分集的全局 index（按章节顺序排列）
8. 返回完整的分集列表

**响应：**

```json
{
  "episodes": [...],
  "stats": {
    "episodeCount": 8,
    "totalScenes": 32,
    "totalDuration": "480s",
    "characterCount": 6,
    "generatedChapters": 3,
    "skippedChapters": 2
  }
}
```

**错误处理：**
- 项目无已确认的小说数据 → 返回 400
- 指定的 chapterId 不存在 → 返回 400
- AI 服务不可用 → 返回 503
- 部分章节生成失败 → 返回已成功的结果 + 失败章节列表

---

### `POST /api/episodes/confirm`

确认分集大纲，更新项目进度。

**请求体：**

```json
{
  "projectId": "clyyy..."
}
```

**流程：**

1. 校验该项目至少有 1 集分集大纲
2. 更新 Project：`step = 3`，`stepLabel = "剧本生成"`
3. 返回更新后的 Project

## 3.5 状态管理（Zustand Store）

```typescript
// store/outline-store.ts

interface Episode {
  id: string
  projectId: string
  chapterId: string
  chapter: { id: string; index: number; title: string | null }
  index: number
  title: string
  synopsis: string
  keyConflict: string | null
  cliffhanger: string | null
  duration: string
  scenes: EpisodeScene[]
  createdAt: string
  updatedAt: string
}

interface EpisodeScene {
  id: string
  episodeId: string
  index: number
  title: string
  summary: string
  duration: string
  characters: string | null  // JSON string
  emotion: string | null
  createdAt: string
  updatedAt: string
}

type GenerateStatus = "idle" | "generating" | "completed" | "error"

interface GenerateProgress {
  current: number          // 当前正在生成第几个章节（从 1 开始）
  total: number            // 总共需要生成的章节数
  currentChapterTitle: string  // 当前正在生成的章节标题
}

interface EpisodeInput {
  chapterId: string
  title: string
  synopsis?: string
  keyConflict?: string
  cliffhanger?: string
  duration?: string
}

interface SceneInput {
  title: string
  summary?: string
  duration?: string
  characters?: string
  emotion?: string
}

interface OutlineState {
  episodes: Episode[]
  generateStatus: GenerateStatus
  generateError: string | null
  generateProgress: GenerateProgress | null

  // 章节筛选
  filterChapterIds: string[]  // 空数组表示"全部"

  // 加载
  fetchEpisodes: (projectId: string, chapterId?: string) => Promise<void>

  // AI 生成
  generateOutline: (
    projectId: string,
    chapterIds?: string[],
    mode?: "skip_existing" | "overwrite"
  ) => Promise<void>

  // 分集 CRUD
  addEpisode: (projectId: string, data: EpisodeInput) => Promise<void>
  updateEpisode: (episodeId: string, data: Partial<EpisodeInput>) => Promise<void>
  deleteEpisode: (episodeId: string) => Promise<void>
  reorderEpisodes: (projectId: string, orderedIds: string[]) => Promise<void>

  // 场景 CRUD
  addScene: (episodeId: string, data: SceneInput) => Promise<void>
  updateScene: (episodeId: string, sceneId: string, data: Partial<SceneInput>) => Promise<void>
  deleteScene: (episodeId: string, sceneId: string) => Promise<void>
  reorderScenes: (episodeId: string, orderedIds: string[]) => Promise<void>

  // 筛选
  setFilterChapterIds: (chapterIds: string[]) => void

  // 确认
  confirmOutline: (projectId: string) => Promise<void>

  // 计算属性
  filteredEpisodes: () => Episode[]
  chapterGenerateStatus: (chapterId: string) => "none" | "generated" | "generating" | "error"

  // 重置
  reset: () => void
}
```

## 3.6 页面布局与交互设计

### 整体布局

页面采用**顶部操作栏 + 章节筛选栏 + 分集列表**的纵向布局：

```
┌──────────────────────────────────────────────────────────────┐
│  分集大纲                                                      │
│  基于小说章节内容，AI 将每个章节拆分为多集短剧大纲                     │
│                                                              │
│  ┌──── 操作栏 ──────────────────────────────────────────────┐ │
│  │  [AI 生成全部大纲 ▾]   [添加一集]                          │ │
│  │                                                          │ │
│  │  ▾ 下拉菜单:                                              │ │
│  │    ├─ 生成全部章节（跳过已生成）                              │ │
│  │    ├─ 生成全部章节（全部重新生成）                             │ │
│  │    └─ 选择章节生成...  → 弹出章节选择弹窗                     │ │

### 交互细节

1. **空状态**
   - 首次进入且无分集数据时，显示空状态引导：
     - 插图 + "尚未生成分集大纲"
     - 展示章节列表预览（来自小说导入的章节数据）
     - 「AI 生成全部大纲」主按钮
     - 「选择章节生成」次要按钮
   - 如果小说未导入（Novel 不存在或 status ≠ "confirmed"），显示提示并引导返回小说导入页

2. **AI 生成进度**
   - 生成按钮变为 loading 状态，显示进度："正在生成 第一章·重生 (1/5)..."
   - 每完成一个章节，立即将结果追加到分集列表中（用户可实时看到）
   - 对应的章节筛选标签状态实时更新（灰色 → loading → 绿色勾）
   - 支持取消生成（中断后续章节，已完成的保留）

3. **章节筛选交互**
   - 筛选标签采用可切换的 Badge/Tag 样式
   - 点击标签切换选中/取消选中
   - 「全部」标签始终在最左侧，点击后取消所有筛选
   - 筛选时分集列表平滑过渡（动画）
   - 筛选标签上的数字实时反映该章节的分集数

4. **分集卡片交互**
   - 卡片顶部显示来源章节 Badge（如"📖 第一章 · 重生"），颜色与筛选标签对应
   - 卡片默认折叠场景列表，点击展开/收起
   - hover 显示编辑和删除按钮
   - 拖拽手柄在卡片左侧
   - 编辑弹窗包含：归属章节（下拉选择）、集标题、剧情摘要、核心冲突、结尾钩子、预估时长
   - 删除需二次确认

5. **场景管理交互**
   - 展开场景列表后，每个场景以紧凑行形式展示
   - 点击场景行可编辑
   - 场景编辑弹窗包含：标题、摘要、时长、角色选择（多选）、情感标签选择
   - 场景支持集内拖拽排序
   - 底部「添加场景」按钮

6. **确认大纲**
   - 底部固定操作栏
   - 至少有 1 集时按钮可用，否则禁用
   - 点击后弹出确认弹窗，显示摘要：总集数、总场景、覆盖章节数
   - 确认后更新项目进度并跳转

## 3.8 边界情况与异常处理

| 场景 | 处理方式 |
|------|----------|
| 小说未导入即进入大纲页 | 显示提示"请先完成小说导入"，提供跳转按钮 |
| 小说已导入但未确认 | 显示提示"请先确认小说导入"，提供跳转按钮 |
| 小说无章节数据 | 显示提示"未检测到章节，请返回小说导入页添加章节" |
| AI 生成超时（> 60s/章） | 显示超时提示 + 重试按钮，已成功的章节保留 |
| AI 返回格式异常 | 降级处理：解析可用部分，标记该章节为"部分生成失败" |
| 多章节生成中途失败 | 已成功的章节保留结果，失败的章节标记为错误，提示用户重试失败章节 |
| 重新生成已有大纲的章节 | 弹窗确认"重新生成将覆盖该章节的现有大纲"，确认后清空旧数据再生成 |
| 删除最后一集 | 允许删除，回到空状态 |
| 分集数为 0 时点击确认 | 按钮禁用，tooltip 提示"至少需要 1 集" |
| 章节被删除但分集仍存在 | 级联删除（Prisma onDelete: Cascade），分集随章节一起删除 |
| 筛选后无匹配分集 | 显示"该章节暂无分集大纲"空状态 |
| 场景角色不在角色列表中 | 允许自由输入，不强制匹配 NovelCharacter |
| 网络断开 | 提示网络错误，保留本地编辑状态 |
| 时长输入非法 | 输入校验，仅允许数字 + "s" 格式 |
| 修改分集归属章节 | 全局 index 自动重新计算 |

## 3.9 与其他模块的数据流

```
模块2(小说导入)                  模块3(分集大纲)                模块4(剧本生成)
     │                                │                           │
     │  Novel                         │                           │
     │  ├─ synopsis ─────────────────►│ (整体脉络)                 │
     │  ├─ characters[] ────────────►│ (角色选择数据源)             │
     │  └─ events[] ────────────────►│ (剧情结构参考)              │
     │                                │                           │
     │  Chapter[]                     │                           │
     │  ├─ chapter[0].content ──────►│ → Episode[0], Episode[1]  │
     │  ├─ chapter[1].content ──────►│ → Episode[2]              │
     │  ├─ chapter[2].content ──────►│ → Episode[3], Episode[4]  │
     │  └─ ...                       │                           │
     │                                │                           │
     │                                │  Episode[]                │
     │                                │  ├─ title ───────────────►│
     │                                │  ├─ synopsis ────────────►│
     │                                │  ├─ keyConflict ─────────►│
     │                                │  ├─ chapterId ───────────►│ (溯源)
     │                                │  └─ scenes[] ────────────►│
     │                                │                           │
     Project                          │                           │
     ├─ platform ────────────────────►│                           │
     └─ duration ────────────────────►│                           │
```

- **向上游**：
  - 从 Novel 获取故事大纲、角色列表、剧情事件
  - 从 Chapter 获取各章节内容（核心输入，按章节维度生成）
  - 从 Project 获取平台和时长配置
- **向下游**：
  - 模块 4（剧本生成）：消费 Episode 和 EpisodeScene 作为剧本的结构蓝图
  - 每个 Episode 保留 `chapterId` 引用，支持溯源到原文章节

- 目标平台：{platform}

   - 短章节（< 2000字）：通常 1 集

3. 每集结尾必须有钩子（cliffhanger），制造悬念
4. 每集拆分为 3-6 个场景
5. 每个场景包含：标题、摘要、预估时长、出场角色、情感标签

7. 情感标签从以下选项中选择：平静、紧张、悲伤、激昂、温馨、愤怒、震惊、心动、悬疑、冲击、感慨、压抑
8. 如有前一集的摘要，确保本章第一集与之自然衔接

## 输出格式
请严格按以下 JSON 格式输出：

{
  "episodes": [
    {
      "title": "第X集 · 标题",
      "synopsis": "本集剧情摘要...",
      "keyConflict": "核心冲突描述",
      "cliffhanger": "结尾钩子描述",
      "duration": "60s",
      "scenes": [
        {
          "title": "场景标题",
          "summary": "场景描述...",
          "duration": "15s",
          "characters": ["角色A", "角色B"],
          "emotion": "紧张"
        }
      ]
    }
  ]
}

注意：不要在 episodes 中包含 index 字段，系统会自动计算全局编号。
```

## 3.11 非功能性需求

| 维度 | 要求 |
|------|------|
| **性能** | 分集列表渲染（含 10 集 × 5 场景）应在 200ms 内完成 |
| **AI 响应** | 单章节生成控制在 45s 内，多章节串行生成时每完成一章即更新 UI |
| **数据安全** | 分集数据仅存本地 SQLite，不上传到云端（AI 调用除外） |
| **可编辑性** | 所有 AI 生成结果均可手动修改，AI 只是辅助而非决定 |
| **拖拽体验** | 拖拽排序应流畅，无明显卡顿，支持触摸设备 |
| **数据一致性** | 删除/排序/章节变更操作后，全局 index 自动重新计算 |
| **渐进式生成** | 多章节生成时，已完成的章节结果立即可见，不必等全部完成 |
| **容错性** | 多章节生成中途失败不影响已成功的结果 |
