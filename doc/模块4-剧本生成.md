# 📝 模块 4：剧本生成

## 4.1 模块定位

基于模块 3（分集大纲）的结构化数据，**以分集（Episode）为单位**，AI 自动为每一集生成完整的结构化剧本。每个剧本包含场景序列，每个场景包含角色台词、旁白、动作描述、情绪标签、BGM 建议和时长估算，为后续分镜生成提供可执行的影视脚本。

**核心设计理念**：一集对应一个剧本（1:1 关系）。分集大纲是剧本的骨架，剧本是骨架的血肉填充。用户可以按需选择「生成全部剧本」或「指定分集」进行生成，已生成的剧本可以跳过或重新生成。剧本必须"可编辑"——AI 只是辅助，用户拥有最终决定权。

**前置依赖**：模块 3（分集大纲）已完成，大纲已确认（`Project.step >= 3`，`Project.stepLabel = "剧本生成"`）。

**输出物**：结构化的剧本数据（Script + ScriptScene + ScriptLine），供模块 5（角色生成）和模块 6（分镜生成）消费。

## 4.2 核心功能拆解

### 4.2.1 分集选择与剧本生成

用户进入剧本生成页面后，系统自动加载已确认的分集大纲列表。用户可以选择生成方式：

#### A. 生成全部剧本

- 点击「AI 生成全部剧本」按钮
- 系统遍历所有分集（Episode），逐集调用 AI 生成剧本
- 如果某些分集已有剧本数据，弹窗提示用户选择：
  - 「跳过已生成」：仅生成尚无剧本的分集
  - 「全部重新生成」：清空所有旧剧本数据，全部重新生成
  - 「取消」

#### B. 选择指定分集生成

- 页面展示分集选择器（Checkbox 列表）
- 用户勾选需要生成剧本的分集
- 点击「生成选中剧本」按钮
- 仅对勾选的分集调用 AI 生成
- 如果选中的分集已有剧本，弹窗提示「重新生成将覆盖该分集的现有剧本」

#### C. 单集生成

- 在分集列表中，每个分集条目右侧有独立的「生成剧本」按钮
- 点击后仅生成该集的剧本
- 适合用户对单集剧本不满意时重新生成

#### 输入数据

| 来源 | 数据 | 用途 |
|------|------|------|
| Episode.title | 集标题 | 剧本标题来源 |
| Episode.synopsis | 剧情摘要 | AI 生成剧本的核心依据 |
| Episode.keyConflict | 核心冲突 | 指导冲突场景的台词设计 |
| Episode.cliffhanger | 结尾钩子 | 指导结尾悬念的呈现方式 |
| Episode.duration | 目标时长 | 控制剧本总时长 |
| Episode.scenes[] | 场景列表 | 剧本场景结构的蓝图 |
| Episode.chapter.content | 来源章节原文 | 提供对白和描写的素材 |
| Novel.synopsis | 故事大纲 | 提供整体剧情脉络 |
| Novel.characters[] | 角色列表 | 确保角色台词风格一致 |
| Project.platform | 目标平台 | 影响台词风格（如抖音更口语化） |
| Project.duration | 目标时长 | 控制节奏 |

#### 生成流程

```
用户选择分集 → 逐集构建 AI Prompt → 调用大模型 → 解析 JSON → 批量写入数据库 → 前端渲染
```

> 多集生成时采用**串行**方式逐集调用（保证上下文连贯），每完成一集即更新 UI，用户可实时看到进度。

#### AI Prompt 设计要点

- **角色**：你是一位资深短剧编剧，擅长将大纲转化为高质量的竖屏短剧剧本
- **核心要求**：
  - 基于分集大纲的场景列表，为每个场景生成详细的剧本内容
  - 每个场景包含多条剧本行（ScriptLine），类型包括：对白（dialogue）、旁白（narration）、动作描述（action）、转场指示（transition）
  - 对白要符合角色性格，口语化、有张力
  - 旁白要简洁有力，推动剧情
  - 动作描述要有画面感，便于后续分镜
  - 每个场景附带 BGM 建议和整体情绪基调
  - 根据目标平台调整风格（抖音更快节奏、更口语化）
  - 考虑前后集的衔接，保持叙事连贯
- **输出格式**：要求 AI 返回严格的 JSON 结构
- **上下文窗口**：传入前一集最后一个场景的剧本内容作为衔接参考

#### 生成状态管理

| 状态 | 说明 | UI 表现 |
|------|------|---------|
| `idle` | 未生成 | 显示空状态 + 生成引导 |
| `generating` | 生成中 | 进度指示器 + 当前正在生成的分集名 + 进度（如 2/8） |
| `completed` | 生成完成 | 展示剧本内容 |
| `error` | 生成失败 | 错误提示 + 重试按钮，已成功的分集保留结果 |

### 4.2.2 剧本列表展示与分集筛选

#### A. 剧本列表

以卡片/手风琴列表形式展示所有分集的剧本。按分集 `index` 排序，每个剧本卡片包含：

| 字段 | 说明 |
|------|------|
| 集序号 | 对应 Episode 的全局编号（第1集、第2集...） |
| 集标题 | 如"第1集 · 重逢" |
| 来源章节 | 该集所属的章节标题（Badge 形式） |
| 剧本状态 | 未生成 / 已生成 / 生成中 |
| 场景数量 | 剧本包含的场景数 |
| 台词行数 | 剧本包含的总行数（对白 + 旁白 + 动作） |
| 预估时长 | 剧本总时长 |
| 涉及角色 | 本集出场的角色列表 |

#### B. 按章节筛选

与模块 3 类似，剧本列表上方提供**章节筛选栏**：

- 展示所有章节的筛选标签（Tag/Badge 形式），支持多选
- 默认「全部」选中，显示所有分集剧本
- 筛选标签上显示该章节下的分集数量

#### C. 剧本生成状态一览

每个分集卡片旁显示生成状态图标：

| 状态 | 图标/样式 | 说明 |
|------|-----------|------|
| 未生成 | 灰色圆点 | 该集尚未生成剧本 |
| 已生成 | 绿色勾 | 该集已有剧本数据 |
| 生成中 | 旋转 loading | 正在生成该集的剧本 |
| 生成失败 | 红色叹号 | 该集生成失败 |

### 4.2.3 剧本编辑器

剧本编辑器是本模块的核心交互组件，用户在此查看和编辑剧本内容。

#### A. 编辑器布局

采用**场景分段式**布局，每个场景作为一个独立的编辑区块：

```
┌─ 场景 1：机场回归 ──────────────────────────────────────┐
│  📍 场景信息：时长 12s | 情绪：感慨 | BGM：轻柔钢琴曲      │
│                                                         │
│  [旁白] 五年后的机场，一切都变了，又好像什么都没变。          │
│  [动作] 林晚秋拖着行李箱走出机场，阳光刺眼，她微微眯眼。     │
│  [对白] 林晚秋：（轻声）"回来了..."                        │
│  [动作] 她深吸一口气，迈步走向出租车站。                     │
│                                                         │
│  [+ 添加行]                                              │
└─────────────────────────────────────────────────────────┘

┌─ 场景 2：宴会重逢 ──────────────────────────────────────┐
│  📍 场景信息：时长 20s | 情绪：震惊 | BGM：紧张弦乐        │
│  ...                                                    │
└─────────────────────────────────────────────────────────┘
```

#### B. 剧本行（ScriptLine）类型

| 类型 | 标识 | 说明 | 显示样式 |
|------|------|------|----------|
| `dialogue` | [对白] | 角色台词 | 角色名 + 括号情绪 + 台词内容，高亮显示 |
| `narration` | [旁白] | 画外音/旁白 | 斜体，灰色背景 |
| `action` | [动作] | 动作/场景描述 | 正常字体，浅蓝背景 |
| `transition` | [转场] | 转场指示 | 居中，粗体，分隔线 |

#### C. 剧本行编辑

每行剧本内容支持：

- **点击编辑**：点击行内容进入编辑模式（inline-edit）
- **类型切换**：下拉选择行类型（对白/旁白/动作/转场）
- **角色选择**：对白类型时，下拉选择说话角色（数据源来自 NovelCharacter）
- **情绪标注**：对白类型时，可选标注角色情绪（如"愤怒"、"哽咽"）
- **时长调整**：每行可单独设置预估时长
- **拖拽排序**：行内拖拽调整顺序
- **删除行**：每行右侧删除按钮
- **添加行**：场景底部「添加行」按钮，或在任意行之间插入

#### D. 场景级操作

- **编辑场景信息**：修改场景标题、整体情绪、BGM 建议、预估时长
- **删除场景**：删除整个场景（含所有行），需二次确认
- **添加场景**：在任意位置插入新场景
- **拖拽排序**：场景间拖拽调整顺序
- **折叠/展开**：场景支持折叠，仅显示标题和摘要信息

### 4.2.4 剧本管理（Script CRUD）

#### A. 剧本与分集的关系

- 一个 Episode 对应一个 Script（1:1 关系）
- Script 创建时自动关联到对应的 Episode
- 删除 Episode 时级联删除其 Script

#### B. 重新生成剧本

- 在已有剧本的分集上点击「重新生成」
- 弹窗确认「重新生成将覆盖现有剧本内容」
- 确认后清空旧 Script 数据，重新调用 AI 生成
- 支持保留用户手动修改的部分（可选，V2 功能）

#### C. 剧本预览

- 提供「预览模式」，以只读的剧本格式展示完整剧本
- 预览模式下隐藏所有编辑控件
- 支持按场景折叠/展开
- 显示完整的时长统计和角色出场统计

---

### 4.2.5 统计信息面板

页面顶部展示全局统计信息：

| 指标 | 计算方式 |
|------|----------|
| 总剧本数 | 已生成 Script 的数量 / Episode 总数 |
| 总场景数 | 所有 Script 下 ScriptScene 总数 |
| 总台词行数 | 所有 ScriptLine 总数 |
| 预估总时长 | 所有 ScriptScene 时长之和 |
| 涉及角色 | 所有对白行中出场角色的去重集合 |
| 生成覆盖 | 已生成剧本的分集数 / 总分集数（如 5/8） |

---

### 4.2.6 确认剧本与流转

用户审阅并调整剧本后，点击底部「确认剧本，进入角色生成」按钮：

1. 校验至少有 1 个分集已生成剧本
2. 更新项目进度：`step = 4`，`stepLabel = "角色生成"`
3. 页面跳转到 `/project/[id]/characters`

## 4.3 数据模型设计

### Script（剧本）

```prisma
model Script {
  id        String   @id @default(cuid())
  projectId String
  project   Project  @relation(fields: [projectId], references: [id], onDelete: Cascade)

  episodeId String   @unique
  episode   Episode  @relation(fields: [episodeId], references: [id], onDelete: Cascade)

  title     String
  status    String   @default("draft")  // "draft" | "generated" | "edited"

  scenes ScriptScene[]

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

### ScriptScene（剧本场景）

```prisma
model ScriptScene {
  id       String @id @default(cuid())
  scriptId String
  script   Script @relation(fields: [scriptId], references: [id], onDelete: Cascade)

  index       Int
  title       String
  description String?
  duration    String   @default("15s")
  emotion     String?
  bgm         String?
  location    String?    // 场景地点（如"机场大厅"、"豪华宴会厅"）

  lines ScriptLine[]

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

### ScriptLine（剧本行）

```prisma
model ScriptLine {
  id      String      @id @default(cuid())
  sceneId String
  scene   ScriptScene @relation(fields: [sceneId], references: [id], onDelete: Cascade)

  index       Int
  type        String    // "dialogue" | "narration" | "action" | "transition"
  character   String?   // 说话角色名（仅 dialogue 类型）
  emotion     String?   // 角色情绪（仅 dialogue 类型，如"愤怒"、"哽咽"）
  content     String    // 台词/旁白/动作描述内容
  duration    String?   // 预估时长（如"3s"）
  parenthetical String? // 括号注释（如"低声"、"转身"）

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

### 关联关系更新

在 `Project` 模型中新增：

```prisma
model Project {
  // ... 现有字段
  novel    Novel?
  episodes Episode[]
  scripts  Script[]
}
```

在 `Episode` 模型中新增：

```prisma
model Episode {
  // ... 现有字段
  script Script?
}
```

> **核心关系**：
> - Project 1:N Script（一个项目有多个剧本）
> - Episode 1:1 Script（一个分集对应一个剧本）
> - Script 1:N ScriptScene（一个剧本有多个场景）
> - ScriptScene 1:N ScriptLine（一个场景有多条剧本行）

### `POST /api/scripts/generate`

AI 生成剧本（支持全部分集或指定分集）。

**请求体：**

```json
{
  "projectId": "clyyy...",
  "episodeIds": ["ep_1", "ep_3"],
  "mode": "skip_existing"
}
```

**参数说明：**

| 参数 | 必填 | 说明 |
|------|------|------|
| projectId | 是 | 项目 ID |
| episodeIds | 否 | 指定分集 ID 列表。为空或不传时，生成所有分集的剧本 |
| mode | 否 | 冲突处理模式：`"skip_existing"`（跳过已生成）/ `"overwrite"`（覆盖重新生成）。默认 `"skip_existing"` |

**流程：**

1. 从数据库读取该项目的 Novel 数据（synopsis、characters）
2. 读取所有 Episode 数据（含 scenes），确定待生成的分集列表
3. 根据 `mode` 处理冲突：
   - `skip_existing`：过滤掉已有 Script 的分集
   - `overwrite`：先删除这些分集的旧 Script 数据
4. 读取 Project 配置（platform、duration）
5. 逐集调用 AI 大模型生成剧本
6. 解析返回的 JSON，批量创建 Script、ScriptScene、ScriptLine
7. 返回完整的剧本列表

**响应：**

```json
{
  "scripts": [...],
  "stats": {
```

     - 展示分集列表预览（来自分集大纲的数据）
     - 「AI 生成全部剧本」主按钮
     - 「选择分集生成」次要按钮

2. **AI 生成进度**
   - 生成按钮变为 loading 状态，显示进度："正在生成 第1集·重逢 (1/8)..."
   - 每完成一集，立即将结果追加到左侧分集导航和右侧编辑器中
   - 对应的分集状态实时更新（灰色 → loading → 绿色勾）

4. **剧本编辑器交互**
   - 右侧编辑器显示当前选中分集的完整剧本
   - 场景以卡片形式纵向排列
   - 每个场景卡片顶部显示场景信息栏（标题、时长、情绪、BGM、地点）
   - 场景内的剧本行以列表形式展示
   - 不同类型的行用不同的背景色和图标区分
   - 对白行显示角色名和情绪标注
   - 所有修改实时保存（debounce 500ms）

5. **行内编辑**
   - 点击行内容进入编辑模式
   - 编辑区域包含：类型选择、角色选择（对白时）、情绪标注、内容输入、时长
   - 按 Enter 保存并退出编辑
   - 按 Esc 取消编辑
   - 支持 Tab 键在行间快速切换

6. **确认剧本**
   - 底部固定操作栏
   - 至少有 1 个已生成剧本时按钮可用，否则禁用
   - 点击后弹出确认弹窗，显示摘要：已生成剧本数、总场景数、总台词行数
   - 确认后更新项目进度并跳转

## 4.8 边界情况与异常处理

| 场景 | 处理方式 |
|------|----------|
| 大纲未确认即进入剧本页 | 显示提示"请先完成分集大纲"，提供跳转按钮 |
| 分集数为 0 | 显示提示"暂无分集大纲数据，请先生成分集大纲"，提供跳转按钮 |
| AI 生成超时（> 90s/集） | 显示超时提示 + 重试按钮，已成功的分集保留 |
| AI 返回格式异常 | 降级处理：解析可用部分，标记该集为"部分生成失败" |
| 多集生成中途失败 | 已成功的分集保留结果，失败的分集标记为错误，提示用户重试 |
| 重新生成已有剧本的分集 | 弹窗确认"重新生成将覆盖该集的现有剧本"，确认后清空旧数据再生成 |
| 对白行未选择角色 | 保存时提示"对白类型需要指定说话角色" |
| 删除最后一个剧本 | 允许删除，回到空状态 |
| 剧本数为 0 时点击确认 | 按钮禁用，tooltip 提示"至少需要 1 个已生成的剧本" |
| 分集被删除但剧本仍存在 | 级联删除（Prisma onDelete: Cascade），剧本随分集一起删除 |
| 编辑中网络断开 | 提示网络错误，保留本地编辑状态，恢复后自动重试保存 |
| 时长输入非法 | 输入校验，仅允许数字 + "s" 格式 |
| 剧本行内容为空 | 保存时提示"内容不能为空" |
| 场景无剧本行 | 允许空场景，但在统计中标记为"空场景" |
| 角色名不在 NovelCharacter 列表中 | 允许自由输入，不强制匹配（用户可能需要新增角色） |

- **向上游**：
  - 从 Episode 获取分集大纲（标题、摘要、冲突、钩子、场景列表）——核心输入
  - 从 Chapter 获取来源章节原文（对白和描写素材）
  - 从 Novel 获取故事大纲和角色列表（保持风格一致）
  - 从 Project 获取平台和时长配置
- **向下游**：
  - 模块 5（角色生成）：消费 ScriptLine 中的角色出场数据，统计角色台词风格和情绪范围
  - 模块 6（分镜生成）：消费 ScriptScene 和 ScriptLine 作为分镜设计的依据
  - 每个 Script 保留 `episodeId` 引用，支持溯源到分集大纲和原文章节

## 4.11 非功能性需求

| 维度 | 要求 |
|------|------|
| **性能** | 剧本编辑器渲染（含 5 场景 × 15 行）应在 300ms 内完成 |
| **AI 响应** | 单集剧本生成控制在 60s 内，多集串行生成时每完成一集即更新 UI |
| **数据安全** | 剧本数据仅存本地 SQLite，不上传到云端（AI 调用除外） |
| **可编辑性** | 所有 AI 生成结果均可手动修改，AI 只是辅助而非决定 |
| **实时保存** | 编辑操作 debounce 500ms 后自动保存，无需手动保存按钮 |
| **拖拽体验** | 场景和行的拖拽排序应流畅，无明显卡顿 |
| **数据一致性** | 删除/排序操作后，index 自动重新计算 |
| **渐进式生成** | 多集生成时，已完成的分集结果立即可见，不必等全部完成 |
| **容错性** | 多集生成中途失败不影响已成功的结果 |
| **撤销支持** | V2 考虑支持编辑撤销（Ctrl+Z），V1 暂不实现 |
