# 🖼️ 模块 6：画面生成

## 6.1 模块定位

基于模块 5（分镜生成）的结构化分镜数据，**以镜头（Shot）为单位**，调用 AI 图像生成 API 为每个分镜镜头生成对应的画面图像。生成的图像将直接用于后续视频合成，是整个 CutGo 工业流水线中**"文字转视觉"**的核心环节。

**核心设计理念**：一张好的画面 = 精准的 Prompt + 准确的角色外貌 + 一致的风格基调。模块 5 已完成"写什么"（Prompt）的工作，模块 6 专注于"生成什么"（Image）。用户既可以「一键批量生成全部画面」，也可以「指定分集/场景/单镜头」按需生成，已生成的镜头可以查看、重新生成或手动替换。画面必须"可管理"——AI 是生产力工具，用户拥有最终替换权。

**前置依赖**：模块 5（分镜生成）已完成，分镜已确认（`Project.step >= 6`，`Project.stepLabel = "画面生成"`）。

**输出物**：
- **单帧**：每个 Shot 对应一张图片（`Shot.imageUrl`），用于视频帧叠放或静态展示
- **首尾帧**：镜头的起始图（`Shot.firstFrameUrl`）和结束图（`Shot.lastFrameUrl`），用于约束视频生成模型的运动区间
- **关键帧**：镜头内多个时间节点的图像（`Shot.keyframes[].imageUrl`），精确描述镜头内的动作变化路径
- 所有图片保存到本地 `public/projects/{projectId}/shots/` 目录，URL 回填到对应 Shot 字段，供模块 7（视频合成）消费。

## 6.2 核心概念

### 生成模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 自动模式 | 直接使用分镜 Prompt 调用 AI 图像生成 | 首次批量生成，快速出图 |
| 手动优化模式 | 用户在 Prompt 基础上调整后再生成 | 对单帧质量有较高要求 |
| 参考图模式 | 上传参考图 + Prompt 结合生成（img2img） | 保持角色/场景一致性 |
| 替换模式 | 用户直接上传本地图片替换 AI 生成结果 | 外部素材整合 |
| **首尾帧模式** | 为每个镜头生成首帧和尾帧两张图像，供视频模型做运动插帧 | 需要视频化运动的镜头 |
| **关键帧模式** | 在一个镜头内标记多个时间节点，逐节点生成关键帧图像 | 镜头内有明显动作/情绪转变 |

### 多宫格预览

多宫格预览是一种**以场景为单位**的快速审阅视图，将同一场景下所有镜头的画面拼合为 N×M 宫格图，方便用户在一屏内整体感受场景的视觉连贯性和画面风格一致性：

| 宫格规格 | 适用场景 |
|----------|----------|
| 2×2（4格） | 短场景（2-4 个镜头） |
| 3×2（6格） | 中等场景（5-6 个镜头） |
| 4×3（12格） | 长场景（7-12 个镜头） |
| 自适应（N×2） | 默认，自动按镜头数量决定列数 |

多宫格图可**一键导出**为单张 PNG，用于创作记录、分享或与客户审稿。

### 图像生成后端

| 后端 | 协议 | 适用场景 | 说明 |
|------|------|----------|------|
| ComfyUI（本地） | HTTP API | 本地高质量生成 | 需要用户本地部署，支持 ControlNet、LoRA |
| Replicate API | REST API | 云端快速出图 | 无需本地 GPU，按量计费 |
| 硅基流动 / 智谱 | REST API | 国内云端低成本 | 适合国内用户 |
| Stability AI | REST API | 高质量商业图像 | SDXL / SD3 模型 |

> 当前阶段优先支持 **ComfyUI（本地）** 和 **硅基流动 API**，其他后端预留接口。

### 画面质量关键参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 分辨率 | 输出图像尺寸 | 576×1024（竖屏 9:16） |
| 采样步数 | 生成迭代步数，越高质量越好但耗时增加 | 25 步 |
| CFG Scale | Prompt 引导强度，越高越忠实 Prompt | 7.0 |
| 采样器 | 图像采样算法 | DPM++ 2M Karras |
| 种子 | 随机种子，固定后可复现结果 | 随机 |
| 负面提示词 | 排除元素 | 来自 Shot.negativePrompt + 全局设置 |

## 6.3 核心功能拆解

### 6.3.1 图像生成任务管理

#### A. 批量生成全部画面

- 点击「AI 生成全部画面」按钮
- 系统遍历所有 Storyboard 下的所有 Shot，批量提交生成任务
- 如果某些镜头已有图像（`Shot.imageUrl` 不为空），弹窗提示用户选择：
  - 「跳过已生成」：仅生成尚无图像的镜头
  - 「全部重新生成」：清空所有旧图像，全部重新生成
  - 「取消」

#### B. 选择指定分集/场景生成

- 页面展示分集选择器（Checkbox 列表）
- 用户勾选需要生成画面的分集或场景
- 点击「生成选中画面」按钮
- 仅对勾选范围内的 Shot 提交生成任务

#### C. 单镜头生成

- 在画面列表中，每个镜头卡片提供「重新生成」按钮
- 点击后仅对该镜头重新提交 AI 生成任务
- 适合对单帧画面不满意时快速重试

#### D. 手动替换

- 每个镜头卡片提供「上传替换」按钮
- 支持上传本地图片（JPG / PNG / WebP）
- 上传后图片裁剪到 9:16 比例，保存到本地 `public/projects/{projectId}/shots/` 目录
- 更新 `Shot.imageUrl` 为新图片路径

#### 输入数据

| 来源 | 数据 | 用途 |
|------|------|------|
| Shot.prompt | 画面描述 Prompt（英文） | 图像生成的核心输入 |
| Shot.negativePrompt | 负面提示词 | 排除不需要的元素 |
| Shot.shotSize | 景别 | 辅助构图参考 |
| Shot.composition | 画面构图（中文） | 生成前供用户理解和修改 Prompt |
| Shot.duration | 镜头时长 | 不影响图像生成，但在列表中展示 |
| Shot.dialogueText | 台词文本 | 不影响图像，用于预览时叠加字幕 |
| AssetCharacter.imageUrl | 角色参考图 | img2img / ControlNet 参考（可选） |
| Project.stylePreset | 视觉风格预设 | 追加到 Prompt 末尾 |
| Project.globalNegPrompt | 全局负面提示词 | 与 Shot 级负面提示词合并 |
| Project.aspectRatio | 画幅比例 | 决定生成分辨率（默认 9:16） |

#### 生成流程

```
用户发起生成 → 构建生成参数 → 提交到生成队列 → 调用 AI API → 保存图片到本地 → 回填 Shot.imageUrl → 前端渲染
```

> 多镜头并发生成（最多 4 个并发任务），每完成一个即更新 UI，用户可实时看到进度。本地 ComfyUI 模式受 GPU 限制，实际并发数取决于显存。

#### 生成状态管理

| 状态 | 说明 | UI 表现 |
|------|------|---------|
| `idle` | 未生成 | 显示空白占位框 + 「生成」引导 |
| `queued` | 排队中 | 进度条 + "排队中 (3/8)" |
| `generating` | 生成中 | 旋转动画 + 进度百分比（如基于步数） |
| `completed` | 生成完成 | 展示图像 |
| `error` | 生成失败 | 错误图标 + 重试按钮，其他镜头不受影响 |
| `replaced` | 手动替换 | 图像正常显示，角标标注"已替换" |

### 6.3.5 首尾帧生成

首尾帧模式是面向**视频化**的核心能力升级。在视频模型（如 Kling、Wan、SVD）将静态图像转化为视频片段时，提供首帧（First Frame）和尾帧（Last Frame）可以**精确控制镜头的起始和结束状态**，大幅提升视频生成的可控性和连贯性。

#### A. 模式说明

| 帧类型 | 说明 | 对应视频模型用途 |
|--------|------|-----------------|
| 首帧（First Frame） | 镜头的起始画面状态 | 图生视频的起始帧输入（img2video start frame） |
| 尾帧（Last Frame） | 镜头的结束画面状态 | 控制视频运动终点，减少随机性（end frame conditioning） |
| 首尾帧（Both） | 同时生成首帧和尾帧 | 完整约束镜头的运动区间 |

#### B. 启用方式

- 在**镜头卡片**上或**详情面板**中，提供「帧模式」切换：`单帧` → `首尾帧` → `关键帧`
- 切换为首尾帧模式后，镜头卡片变为双列布局：左侧首帧 + 右侧尾帧

```
┌─ 镜头 #2 ── [单帧 | 首尾帧✓ | 关键帧] ──────────────┐
│                                                      │
│  ┌──────────────┐    ┌──────────────┐               │
│  │              │    │              │               │
│  │   首  帧      │    │   尾  帧      │               │
│  │  (起始状态)   │    │  (结束状态)   │               │
│  │              │    │              │               │
│  └──────────────┘    └──────────────┘               │
│  [🔄 重生成] [↑ 替换]  [🔄 重生成] [↑ 替换]          │
│                                                      │
│  近景  推进  3s                                        │
└──────────────────────────────────────────────────────┘
```

#### C. Prompt 设计差异

| 帧 | Prompt 策略 | 示例追加词 |
|----|-------------|------------|
| 首帧 | 在原始 Prompt 基础上追加"画面开始状态"描述 | `at the beginning of the shot, initial pose` |
| 尾帧 | 在原始 Prompt 基础上追加"画面结束状态"描述 | `at the end of the shot, final pose, motion complete` |

用户可以在详情面板中分别为首帧和尾帧**单独编辑 Prompt**，也可以使用系统自动生成的差异描述。

- 单帧（默认）存储路径：`public/projects/{projectId}/shots/{shotId}.png`

### 6.3.6 关键帧标记与管理

关键帧（Keyframe）是在**一个镜头内**人为标记的重要时间节点。当一个镜头内包含明显的动作变化（如角色从站立到跌倒）、情绪转变（如从愤怒到落泪）或场景切换时，用户可以添加关键帧，为每个节点单独生成一张画面，帮助视频模型更准确地还原镜头内的运动轨迹。

#### A. 关键帧的概念

```
镜头时间轴：
0s ─────────────────────────────────── 5s
│                                        │
[首帧]          [关键帧1]          [尾帧]
 0s               2.5s               5s
 林晚秋          林晚秋抬头            林晚秋
 低头沉思          望向远方              微笑
```

- 一个镜头可以有 **0 到 N 个关键帧**（建议不超过 3 个，视频模型支持有限）
- 每个关键帧有**时间位置**（0~镜头时长秒数）和**独立 Prompt**
- 关键帧生成的图像与首尾帧一样，用于约束视频生成模型

#### B. 添加关键帧

- 在详情面板的时间轴区域点击「+ 添加关键帧」
- 拖拽时间轴上的关键帧标记点调整时间位置
- 每个关键帧可单独输入 Prompt（或由 AI 基于时间位置自动推断）

```
┌─── 镜头详情 #3 · 关键帧模式 ─────────── [✕] ──┐
│                                               │
│  ── 镜头时间轴 (5s) ──────────────────────── │
│  ┌─────────────────────────────────────────┐  │
│  │  [首帧]      [KF1 2.5s]       [尾帧]    │  │
│  │   ◆           ◆                  ◆      │  │
│  │   0s                             5s     │  │
│  │   ────────────●──────────────────       │  │
│  └─────────────────────────────────────────┘  │
│  [+ 添加关键帧]                                │
│                                               │
│  ── 首帧 (0s) ─────── [🔄] [↑] ─────────── │
│  ┌───┐  林晚秋低头沉思，双手握紧包带              │
│  │图 │  Prompt: [Close-up, downward gaze...] │
│  └───┘                                       │
│                                               │
│  ── 关键帧 1 (2.5s) ── [🔄] [↑] [🗑] ────── │
│  ┌───┐  林晚秋抬头，眼中闪过一丝坚定              │
│  │图 │  Prompt: [Close-up, looking up...]    │
│  └───┘                                       │
│                                               │
│  ── 尾帧 (5s) ─────── [🔄] [↑] ─────────── │
│  ┌───┐  林晚秋微笑，深吸一口气                   │
│  │图 │  Prompt: [Close-up, slight smile...]  │
│  └───┘                                       │
│                                               │
│  [← 上一镜头]                    [下一镜头 →]  │
└───────────────────────────────────────────────┘
```

#### C. 关键帧 Prompt 辅助

- **AI 推断**：点击「AI 推断关键帧」，系统根据首帧 Prompt、尾帧 Prompt 和时间位置，用 AI 插值生成中间关键帧的 Prompt 描述
- **快捷复制**：从首帧/尾帧 Prompt 复制后手动修改是最常见的工作流

#### D. 关键帧数据存储

- 关键帧图像路径：`public/projects/{projectId}/shots/{shotId}_kf{index}.png`（如 `_kf0.png`、`_kf1.png`）
- 关键帧元数据存储在 `Shot.keyframes` 字段（JSON）

#### E. 镜头卡片上的关键帧标识

- 卡片底部展示关键帧数量角标：如 `[KF: 3]` 表示该镜头有 3 个关键帧
- 点击角标快速展开关键帧缩略图列表（水平滚动）

### 6.3.7 多宫格预览

多宫格预览允许用户在**一屏内**快速审阅整个场景或整个分集的画面风格连贯性，是提效版的「一镜到底」审稿工具。

#### A. 触发方式

| 入口 | 触发 | 说明 |
|------|------|------|
| 场景分组标题右侧 | 点击「宫格预览」图标 | 预览该场景所有镜头的多宫格图 |
| 操作栏 | 下拉「多宫格预览」 | 选择范围（当前分集 / 全部）后预览 |
| 时间线视图 | 点击「导出宫格图」 | 将当前视图导出为多宫格图片 |

#### B. 宫格布局规则

| 镜头数量 | 布局 | 单格尺寸（基于 576×1024 源图） |
|----------|------|-------------------------------|
| 1-2 | 1×2 水平排列 | 288×512 |
| 3-4 | 2×2 | 288×512 |
| 5-6 | 3×2 | 192×341 |
| 7-9 | 3×3 | 192×341 |
| 10-12 | 4×3 | 144×256 |
| >12 | 自动分页（每页最多 12 格） | 144×256 |

> 多宫格图的整体比例由布局自动决定，**不强制维持 9:16**，以便在显示器上平铺查看。

#### C. 宫格内信息叠加

每个格子上叠加轻量元信息（可在预览时切换显示/隐藏）：

```
┌──────────────────────────────────────────────────────────────────────┐
│  场景 2：宴会重逢  ·  4 镜头  ·  宫格预览             [导出] [关闭]  │
│                                                                      │
│  ┌───────────────────┐  ┌───────────────────┐                       │
│  │                   │  │                   │                       │
│  │     镜头 #1        │  │     镜头 #2        │                       │
│  │                   │  │                   │                       │
│  │   全景 · 6s        │  │  特写 · 2s         │                       │
│  └───────────────────┘  └───────────────────┘                       │
│  ┌───────────────────┐  ┌───────────────────┐                       │
│  │                   │  │                   │                       │
│  │     镜头 #3        │  │     镜头 #4        │                       │
│  │                   │  │                   │                       │
│  │  中近景 · 4s        │  │   近景 · 4s        │                       │
│  └───────────────────┘  └───────────────────┘                       │
│                                                                      │
│  [显示信息叠加 ✓]  [显示字幕 ✓]  [导出为 PNG]  [导出为 PDF]           │
└──────────────────────────────────────────────────────────────────────┘
```

**叠加信息选项**（可分别开关）：
- 镜头序号（如 `#1`）
- 景别 + 时长（如 `全景 · 6s`）
- 台词/字幕文本（叠加在画面底部，仿字幕效果）
- 首尾帧标记（左上角 `F` 首帧 / `L` 尾帧 / `KF` 关键帧小角标）

#### D. 分集多宫格

选择范围为「当前分集」时，系统按场景分页生成多宫格，最终合并为一张大图（或分页 PDF）：

```
第1集 · 重逢  ─────────────────────────────────────────────────────
  场景1·机场回归     场景2·宴会重逢       场景3·走廊对峙
  [#1][#2][#3]     [#1][#2][#3][#4]     [#1][#2]
```

#### E. 导出格式

| 格式 | 说明 |
|------|------|
| PNG | 单张高清宫格图（按场景） |
| PDF | 多页 PDF，每页一个场景，适合发给团队审稿 |
| JSON | 导出宫格图的镜头元数据（用于对接其他工具） |

#### F. 点击交互

- 点击宫格中的某一帧：高亮并在右侧显示该镜头的详情面板（或底部 Tooltip）
- 双击：跳转到该镜头的画面详情编辑页面
- 宫格预览为**只读模式**，不支持直接编辑

## 6.4 数据模型设计

### Shot 模型扩展

在现有 `Shot` 模型基础上，无需新增独立表，通过扩展字段支持画面生成所需信息：

```prisma
model Shot {
  id           String     @id @default(cuid())
  storyboardId String
  storyboard   Storyboard @relation(fields: [storyboardId], references: [id], onDelete: Cascade)

  index           Int
  shotSize        String
  cameraMovement  String    @default("static")
  cameraAngle     String    @default("eye_level")
  composition     String
  prompt          String
  negativePrompt  String?
  duration        String    @default("3s")

  // ── 单帧画面（默认模式）──────────────────────────────
  imageUrl        String?           // 当前使用的图片 URL（相对路径）
  imageStatus     String  @default("idle")
                                    // "idle" | "queued" | "generating" | "completed" | "error" | "replaced"
  imageHistory    String?           // JSON: 历史生成图片 URL 列表（最多保留 3 张）
  imageSeed       Int?              // 生成时使用的随机种子
  imageSteps      Int?              // 生成时使用的采样步数
  imageCfgScale   Float?            // 生成时使用的 CFG Scale
  imageModel      String?           // 生成时使用的模型名称
  imageGeneratedAt DateTime?        // 图片生成时间
  imageError      String?           // 最近一次生成失败的错误信息

  // ── 帧模式控制 ────────────────────────────────────────
  frameMode       String  @default("single")
                                    // "single" | "first_last" | "keyframe"

  // ── 首尾帧（frameMode = "first_last" 时启用）──────────
  firstFrameUrl    String?           // 首帧图片 URL
  firstFramePrompt String?           // 首帧独立 Prompt（为空则使用 Shot.prompt）
  firstFrameStatus String  @default("idle")
  lastFrameUrl     String?           // 尾帧图片 URL
  lastFramePrompt  String?           // 尾帧独立 Prompt
  lastFrameStatus  String  @default("idle")

  // ── 关键帧（frameMode = "keyframe" 时启用）────────────
  keyframes        String?           // JSON: Keyframe[] 数组
                                    // [{index, timeOffset, prompt, imageUrl, status}, ...]

  scriptLineIds   String?
  dialogueText    String?
  actionNote      String?

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

### ImageTask（生成任务队列）

```prisma
model ImageTask {
  id        String  @id @default(cuid())
  projectId String
  project   Project @relation(fields: [projectId], references: [id], onDelete: Cascade)

  shotId    String
  shot      Shot   @relation(fields: [shotId], references: [id], onDelete: Cascade)

  status    String  @default("queued")  // "queued" | "running" | "completed" | "failed" | "cancelled"
  backend   String                       // "comfyui" | "siliconflow" | "replicate" | "stability"
  frameType String  @default("single")  // "single" | "first" | "last" | "keyframe"
  keyframeIndex Int?                     // 关键帧索引（frameType = "keyframe" 时有值）
  prompt    String                       // 实际发送给 API 的最终 Prompt
  negativePrompt String?
  params    String?                      // JSON: 生成参数（steps, cfg, seed, model 等）

  progress  Int     @default(0)          // 0-100 进度百分比
  resultUrl String?                      // 生成成功后的图片 URL
  error     String?                      // 失败原因

  startedAt   DateTime?
  completedAt DateTime?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}
```

### Project 模型扩展

```prisma
model Project {
  // ... 现有字段
  imageTasks     ImageTask[]

  // 画面生成全局配置
  imageBackend      String   @default("comfyui")   // 使用的图像生成后端
  imageModel        String?                          // 默认模型
  imageSteps        Int      @default(25)            // 默认采样步数
  imageCfgScale     Float    @default(7.0)           // 默认 CFG Scale
  imageResolution   String   @default("576x1024")    // 默认输出分辨率
  imageStyleAppend  String?                          // 全局风格追加词
}
```

> **核心关系**：
>
> - Project 1:N ImageTask（一个项目有多个生成任务记录）
> - Shot 1:N ImageTask（一个镜头可能有多次生成记录）
> - Shot.imageUrl 始终指向当前正在使用的图片
> - Shot.imageHistory 存储历史版本，支持回滚

### `POST /api/images/generate`

提交画面生成任务（支持批量）。

**请求体：**

```json
{
  "projectId": "clyyy...",
  "shotIds": ["clsh1...", "clsh2..."],
  "mode": "skip_existing",
  "overrideParams": {
    "steps": 30,
    "cfgScale": 7.5,
    "seed": 12345
  }
}
```

**参数说明：**

| 参数 | 必填 | 说明 |
|------|------|------|
| projectId | 是 | 项目 ID |
| shotIds | 否 | 指定镜头 ID 列表。为空时生成项目下所有镜头 |
| episodeIds | 否 | 指定分集 ID 列表（与 shotIds 二选一） |
| mode | 否 | 冲突处理：`"skip_existing"` / `"overwrite"`，默认 `"skip_existing"` |
| overrideParams | 否 | 覆盖项目级默认生成参数 |

**流程：**

1. 确定待生成的 Shot 列表
2. 根据 `mode` 过滤/清理已有图像
3. 为每个 Shot 创建 `ImageTask` 记录（状态 `queued`）
4. 按并发数依次执行生成任务：
   a. 构建最终 Prompt（Shot.prompt + Project.imageStyleAppend）
   b. 合并负面提示词（Shot.negativePrompt + Project.globalNegPrompt）
   c. 调用对应后端 API
   d. 保存图片到 `public/projects/{projectId}/shots/{shotId}.png`
   e. 更新 Shot.imageUrl、Shot.imageStatus、Shot.imageSeed 等字段
   f. 更新 ImageTask 状态为 `completed`
5. 返回任务列表

**响应：**

```json
{
  "tasks": [
    {
      "id": "task1...",
      "shotId": "clsh1...",
      "status": "queued"
    }
  ],
  "totalQueued": 12,
  "estimatedSeconds": 300
}
```

### `POST /api/images/replace/:shotId`

**请求类型**：`multipart/form-data`

**请求参数：**

| 参数 | 必填 | 说明 |
|------|------|------|
| file | 是 | 图片文件（JPG/PNG/WebP，最大 10MB） |

1. 接收上传的图片文件

### `DELETE /api/images/:shotId`

删除指定镜头的当前画面，重置为未生成状态。

**流程：**

### 交互细节

1. **空状态**
   - 首次进入且无画面数据时，显示空状态引导：
     - 插图 + "尚未生成画面"
     - 展示分镜 Prompt 预览（来自分镜数据）
     - 「AI 生成全部画面」主按钮
     - 「⚙ 生成设置」引导用户配置后端
   - 如果分镜未确认（`Project.step < 6`），显示提示并引导返回分镜生成页
   - 如果未配置有效的图像生成后端，提示「请先配置画面生成设置」并高亮设置按钮

2. **生成进度**
   - 生成按钮变为 loading 状态，显示进度："正在生成 12/48..."
   - 每完成一个镜头，对应的空白占位框以动画淡入方式显示生成的图片
   - 队列浮层实时更新（排队/运行/完成数量）
   - 支持暂停/取消（暂停：不发起新任务，已运行的等待完成；取消：清空队列，正在运行的继续到完成）

3. **图像卡片悬浮操作**
   - 鼠标悬浮在已生成图像上时，显示操作按钮层：
     - 🔍 查看详情（打开详情面板）
     - 🔄 重新生成
     - ↑ 替换
     - ⬇ 下载
   - 未生成的占位框显示「▶ 生成」按钮

4. **右键上下文菜单**
   - 右键点击已生成图像弹出菜单：
     - 查看详情
     - 重新生成
     - 替换为本地图片
     - 在前方插入镜头（跳转到分镜模块）
     - 下载图片
     - 删除画面（重置为未生成）

5. **批量操作**
   - 按住 Shift 点击可多选镜头卡片
   - 选中多张后顶部出现批量操作栏：「重新生成选中项」「删除选中画面」
   - 批量操作前弹窗确认

6. **生成失败重试**
   - 失败的镜头显示红色叹号 + 错误信息（hover 展示）
   - 单击错误状态的卡片展示详细错误原因（如 API 超时、网络错误）
   - 提供「重试」按钮，也可在批量操作中「重试所有失败项」

7. **确认画面**
   - 底部固定操作栏
   - 至少有 1 个已生成画面时按钮可用，否则禁用
   - 点击后弹出确认弹窗，显示摘要：已生成画面数、存储大小、未生成镜头数（如有）
   - 确认后更新项目进度并跳转

## 6.9 后端集成：ComfyUI

ComfyUI 是本地部署的 AI 图像生成工作流，通过 HTTP API 和 WebSocket 与 CutGo 集成。

### 连接方式

| 接口 | 说明 |
|------|------|
| `GET http://localhost:8188/system_stats` | 检测 ComfyUI 是否运行 |
| `POST http://localhost:8188/prompt` | 提交生成任务（返回 `prompt_id`） |
| `WebSocket ws://localhost:8188/ws` | 订阅任务执行进度事件 |
| `GET http://localhost:8188/history/{prompt_id}` | 查询任务历史（获取输出文件名） |
| `GET http://localhost:8188/view?filename=xxx` | 下载生成的图片 |

### Workflow 设计

CutGo 为 ComfyUI 预置标准 workflow JSON（存放于 `lib/comfyui/workflows/`），根据参数动态注入 Prompt、分辨率等：

```
workflows/
├── text2image-basic.json        # 基础文生图 workflow
├── text2image-high-quality.json # 高质量文生图（更多步数）
└── img2image-reference.json     # 参考图 + Prompt workflow（可选）
```

### 进度事件处理

ComfyUI 通过 WebSocket 推送进度事件：

```typescript
// WebSocket 事件类型
type ComfyUIEvent =
  | { type: "execution_start"; data: { prompt_id: string } }
  | { type: "progress"; data: { value: number; max: number; prompt_id: string } }
  | { type: "executed"; data: { prompt_id: string; output: { images: Array<{ filename: string }> } } }
  | { type: "execution_error"; data: { prompt_id: string; exception_message: string } }
```

## 6.10 边界情况与异常处理

| 场景 | 处理方式 |
|------|----------|
| 分镜未确认即进入画面生成页 | 显示提示"请先完成分镜生成"，提供跳转按钮 |
| 未配置图像生成后端 | 显示引导提示 + 高亮「⚙ 生成设置」按钮 |
| ComfyUI 未启动/连接失败 | 检测失败时提示"ComfyUI 未运行，请先启动本地 ComfyUI 服务" |
| ComfyUI VRAM 不足 | 捕获错误，提示"GPU 显存不足，建议降低分辨率或采样步数" |
| 云端 API Key 无效 | 提示"API Key 无效，请在生成设置中更新"，不发起生成 |
| 云端 API 余额不足 | 捕获 402 响应，提示"账户余额不足，请充值后重试" |
| 生成任务超时（>120s） | 标记为失败，提示"生成超时"，提供重试按钮 |
| 图片保存磁盘空间不足 | 捕获 IO 错误，提示"磁盘空间不足，请释放空间后重试" |
| 上传替换的图片格式不支持 | 前端校验，仅允许 JPG/PNG/WebP，超出限制时提示 |
| 上传替换的图片超过 10MB | 前端校验，提示"图片大小不超过 10MB" |
| 分镜 Prompt 为空 | 跳过该镜头生成，提示"镜头 #X 的 Prompt 为空，已跳过" |
| 生成中途关闭页面/刷新 | 任务状态已持久化到数据库，重新进入页面时自动恢复任务状态 |
| ComfyUI 中途断开 | 标记正在运行的任务为失败，重新连接后可手动重试 |
| 图片文件损坏/无法加载 | 图片区域显示错误图标，提供「重新生成」按钮 |
| 批量生成时部分任务失败 | 成功的保留结果，失败的标记错误，任务队列显示失败数量，提供「重试所有失败项」 |
| 生成历史超过 3 条 | 自动删除最旧的历史图片文件，保留最新 3 条 |
| 镜头无对应分镜数据 | 不应出现（模块依赖保证），如发生则提示并引导重新生成分镜 |
| 点击确认时存在大量未生成镜头 | 弹窗额外提示"尚有 X 个镜头未生成画面，是否继续？" |
| 视频合成需要的图片被删除 | 合成前校验，发现缺失则提示并跳转回画面生成页重新生成 |
| 切换帧模式时已有生成数据 | 切换不删除已有图片，仅切换展示视图；切换回「单帧」后首尾帧/关键帧数据隐藏但保留 |
| 首尾帧尾帧 Prompt 为空 | AI 自动根据 Shot.prompt + 镜头运动类型（推进→动作末尾状态）推断尾帧 Prompt |
| 关键帧 timeOffset 超出镜头时长 | 前端限制输入范围为 (0, Shot.duration)，超出时自动 clamp |
| 关键帧数量超过 3 个 | 允许超过，但提示"超过 3 个关键帧可能超出视频模型输入限制" |
| AI 推断关键帧 Prompt 失败 | 降级为复制首帧 Prompt，提示用户手动修改 |
| 多宫格预览中存在未生成镜头 | 用灰色占位格替代，叠加文字"未生成" |
| 宫格合成时 sharp 依赖缺失 | 降级为前端 CSS Grid 预览（仅在浏览器中展示，不支持导出） |
| 宫格图镜头数量超过 12 | 自动分页，每页 12 格，PDF 导出时每页一页 |

```

     │     ├─ dialogueText ─────────►│ (字幕预览叠加)            │

     │                              │                          │
```

- **向上游**：
  - 从 Shot 获取 Prompt / negativePrompt / composition / duration / dialogueText——核心生成输入
  - 从 AssetCharacter 获取角色外貌描述和参考图——提升角色一致性
  - 从 Project 获取风格预设、全局负面提示词、画幅比例——调性统一
- **向下游**：
  - 模块 7（视频合成）：消费 Shot.imageUrl 作为视频帧素材
  - 消费 Shot.dialogueText 叠加字幕
  - 消费 Shot.duration 计算每帧显示时长

## 6.12 非功能性需求

| 维度 | 要求 |
|------|------|
| **生成速度（本地 ComfyUI）** | 单张 576×1024 图像，25步，RTX 3080 下约 8-15s |
| **生成速度（云端 API）** | 单张约 5-20s，取决于 API 提供商 |
| **并发控制** | 云端模式最多 4 并发，本地 ComfyUI 默认 1 串行 |
| **图片质量** | 输出图像清晰，无明显伪影，符合 Prompt 描述 |
| **存储效率** | 图片以 PNG 格式保存，单张约 1-3MB；批量生成后可手动清理历史版本 |
| **UI 响应性** | 生成中 UI 不卡顿，进度更新实时反映到卡片状态 |
| **断点续传** | 进度状态持久化，关闭页面后重新进入可继续查看/恢复未完成任务 |
| **错误可见性** | 每个失败任务有清晰的错误原因展示，方便用户排查 |
| **数据安全** | 图片文件保存在本地 `public/projects/` 目录，不自动上传云端 |
| **可替换性** | 所有 AI 生成结果均可手动上传图片替换，保障最终创作控制权 |
| **历史版本** | 每个镜头保留最多 3 个历史版本，可随时回滚 |
| **响应式布局** | 三栏布局在不同屏幕宽度下自适应（≥1440px 三栏 / 1024-1439px 双栏 / <1024px 单栏） |
| **竖屏优化** | 默认输出 9:16 比例图像，预览区域始终保持竖屏比例展示 |
