# 模块 6：画面生成

## 6.1 模块定位

基于模块 5（分镜生成）的结构化分镜数据，**以镜头（Shot）为单位**，调用 AI 图像生成 API 为每个分镜镜头生成对应的画面图像。生成的图像将直接用于后续视频合成，是整个 CutGo 工业流水线中**"文字转视觉"**的核心环节。

**核心设计理念**：以 Prompt 为核心驱动，通过标准化接口对接图像生成能力，具体后端实现可插拔替换。用户既可以「一键批量生成全部画面」，也可以「单镜头按需生成」，已生成的镜头可以查看、重新生成或手动替换。

**前置依赖**：模块 5（分镜生成）已完成，分镜已确认（`Project.step >= 6`，`Project.stepLabel = "画面生成"`）。

**输出物**：每个 Shot 对应的图片文件（保存到本地 `public/projects/{projectId}/shots/` 目录），并将图片 URL 回填到 `Shot.imageUrl` 字段，供模块 7（视频合成）消费。

---

## 6.2 核心概念

### 生成模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 自动模式 | 直接使用分镜 Prompt 调用 AI 图像生成 | 首次批量生成，快速出图 |
| 手动优化模式 | 用户在 Prompt 基础上调整后再生成 | 对单帧质量有较高要求 |
| 替换模式 | 用户直接上传本地图片替换 AI 生成结果 | 外部素材整合 |

### 图像生成后端接口

当前阶段通过统一的 `ImageBackend` 接口对接图像生成能力，**具体后端实现预留扩展**，暂以占位实现为主：

```typescript
interface ImageBackend {
  generate(params: GenerateParams): Promise<GenerateResult>
  checkHealth(): Promise<boolean>
}

interface GenerateParams {
  prompt: string
  negativePrompt?: string
  resolution: string   // 如 "576x1024"
  seed?: number
}

interface GenerateResult {
  imageData: Buffer    // 图片二进制数据
  seed: number         // 实际使用的 seed（随机时由后端返回）
}
```

> 后续对接 ComfyUI、硅基流动等具体后端时，只需实现此接口即可，不影响上层业务逻辑。

### 画面生成关键参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 分辨率 | 输出图像尺寸 | 576×1024（竖屏 9:16） |
| 种子 | 随机种子，固定后可复现结果 | 随机 |
| 负面提示词 | 排除元素 | 来自 Shot.negativePrompt + 全局设置 |

## 6.3 核心功能拆解

### 6.3.1 图像生成任务管理

#### A. 批量生成全部画面

- 点击「AI 生成全部画面」按钮
- 系统遍历所有 Storyboard 下的所有 Shot，批量提交生成任务
- 如果某些镜头已有图像（`Shot.imageUrl` 不为空），弹窗提示用户选择：
  - 「跳过已生成」：仅生成尚无图像的镜头
  - 「全部重新生成」：清空所有旧图像，全部重新生成
  - 「取消」

#### B. 选择指定分集生成

- 页面展示分集选择器（Checkbox 列表）
- 用户勾选需要生成画面的分集
- 点击「生成选中画面」按钮
- 仅对勾选范围内的 Shot 提交生成任务

#### C. 单镜头生成

- 在画面列表中，每个镜头卡片提供「重新生成」按钮
- 点击后仅对该镜头重新提交 AI 生成任务
- 适合对单帧画面不满意时快速重试

#### D. 手动替换

- 每个镜头卡片提供「上传替换」按钮
- 支持上传本地图片（JPG / PNG / WebP）
- 上传后保存到本地 `public/projects/{projectId}/shots/` 目录
- 更新 `Shot.imageUrl` 为新图片路径

#### 输入数据

| 来源 | 数据 | 用途 |
|------|------|------|
| Shot.prompt | 画面描述 Prompt（英文） | 图像生成的核心输入 |
| Shot.negativePrompt | 负面提示词 | 排除不需要的元素 |
| Shot.shotSize | 景别 | 辅助构图参考 |
| Shot.composition | 画面构图（中文） | 生成前供用户理解和修改 Prompt |
| Shot.duration | 镜头时长 | 不影响图像生成，但在列表中展示 |
| Shot.dialogueText | 台词文本 | 不影响图像，用于预览时叠加字幕 |
| Project.stylePreset | 视觉风格预设 | 追加到 Prompt 末尾 |
| Project.globalNegPrompt | 全局负面提示词 | 与 Shot 级负面提示词合并 |
| Project.aspectRatio | 画幅比例 | 决定生成分辨率（默认 9:16） |

#### 生成流程

```
用户发起生成 → 构建生成参数 → 提交到生成队列 → 调用 ImageBackend.generate() → 保存图片到本地 → 回填 Shot.imageUrl → 前端渲染
```

> 多镜头并发生成（最多 4 个并发任务），每完成一个即更新 UI，用户可实时看到进度。

#### 生成状态管理

| 状态 | 说明 | UI 表现 |
|------|------|---------|
| `idle` | 未生成 | 显示空白占位框 + 「生成」引导 |
| `queued` | 排队中 | 进度条 + "排队中 (3/8)" |
| `generating` | 生成中 | 旋转动画 |
| `completed` | 生成完成 | 展示图像 |
| `error` | 生成失败 | 错误图标 + 重试按钮，其他镜头不受影响 |
| `replaced` | 手动替换 | 图像正常显示，角标标注"已替换" |

model ImageTask {

### `POST /api/images/generate`

提交画面生成任务（支持批量）。

**请求体：**

```json
{
  "projectId": "clyyy...",
  "shotIds": ["clsh1...", "clsh2..."],
  "mode": "skip_existing"
}
```

**参数说明：**

| 参数 | 必填 | 说明 |
|------|------|------|
| projectId | 是 | 项目 ID |
| shotIds | 否 | 指定镜头 ID 列表。为空时生成项目下所有镜头 |
| episodeIds | 否 | 指定分集 ID 列表（与 shotIds 二选一） |
| mode | 否 | 冲突处理：`"skip_existing"` / `"overwrite"`，默认 `"skip_existing"` |

**流程：**

1. 确定待生成的 Shot 列表
2. 根据 `mode` 过滤/清理已有图像
3. 为每个 Shot 创建 `ImageTask` 记录（状态 `queued`）
4. 按并发数依次执行生成任务：
   a. 构建最终 Prompt（Shot.prompt + Project.imageStyleAppend）
   b. 合并负面提示词（Shot.negativePrompt + Project.globalNegPrompt）
   c. 调用 `ImageBackend.generate()`
   d. 保存图片到 `public/projects/{projectId}/shots/{shotId}.png`
   e. 更新 Shot.imageUrl、Shot.imageStatus、Shot.imageSeed 等字段
   f. 更新 ImageTask 状态为 `completed`
5. 返回任务列表

**响应：**

```json
{
  "tasks": [
    {
      "id": "task1...",
      "shotId": "clsh1...",
      "status": "queued"
    }
  ],
  "totalQueued": 12
}
```

---

### `POST /api/images/generate/:shotId`

重新生成单个镜头的画面。

**请求体：**

```json
{
  "prompt": "可选：覆盖当前 Prompt，不传则使用 Shot.prompt",
  "negativePrompt": "可选：覆盖负面提示词",
  "seed": null
}
```

**流程：**

1. 将旧图像 URL 追加到 `Shot.imageHistory`（最多保留 3 条）
2. 更新 Shot.imageStatus 为 `queued`
3. 创建 ImageTask，立即执行生成
4. 生成完成后更新 Shot.imageUrl、Shot.imageStatus

---

### `POST /api/images/replace/:shotId`

上传本地图片替换指定镜头的画面。

**请求类型**：`multipart/form-data`

**请求参数：**

| 参数 | 必填 | 说明 |
|------|------|------|
| file | 是 | 图片文件（JPG/PNG/WebP，最大 10MB） |

**流程：**

1. 接收上传的图片文件
2. 保存到 `public/projects/{projectId}/shots/{shotId}_replaced.png`
3. 将旧图像 URL 追加到 `Shot.imageHistory`
4. 更新 `Shot.imageUrl` 和 `Shot.imageStatus = "replaced"`

**响应：**

```json
{
  "shotId": "clsh1...",
  "imageUrl": "/projects/clyyy/shots/clsh1_replaced.png",
  "imageStatus": "replaced"
}
```

---

### `DELETE /api/images/:shotId`

删除指定镜头的当前画面，重置为未生成状态。

**流程：**

1. 从磁盘删除图片文件
2. 清空 `Shot.imageUrl`，设置 `Shot.imageStatus = "idle"`
3. 保留 `Shot.imageHistory`（不清空历史记录）

---

### `DELETE /api/images/tasks/:taskId`

取消排队中的生成任务（已在运行中的任务无法取消）。

## 6.6 状态管理（Zustand Store）

## 6.9 图像生成后端接口抽象

所有图像生成能力通过统一接口抽象，方便后续扩展不同后端：

```typescript
// lib/image-backend/types.ts

export interface GenerateParams {
  prompt: string
  negativePrompt?: string
  resolution: string   // 格式: "widthxheight"，如 "576x1024"
  seed?: number
}

export interface GenerateResult {
  imageData: Buffer
  seed: number
}

export interface ImageBackend {
  name: string
  generate(params: GenerateParams): Promise<GenerateResult>
  checkHealth(): Promise<boolean>
}
```

## 6.10 边界情况与异常处理

| 场景 | 处理方式 |
|------|----------|
| 分镜未确认即进入画面生成页 | 显示提示"请先完成分镜生成"，提供跳转按钮 |
| 图像生成后端未配置/不可用 | 显示引导提示"图像生成后端尚未配置"，不发起生成 |
| 生成任务超时（>120s） | 标记为失败，提示"生成超时"，提供重试按钮 |
| 图片保存磁盘空间不足 | 捕获 IO 错误，提示"磁盘空间不足，请释放空间后重试" |
| 上传替换的图片格式不支持 | 前端校验，仅允许 JPG/PNG/WebP |
| 上传替换的图片超过 10MB | 前端校验，提示"图片大小不超过 10MB" |
| 分镜 Prompt 为空 | 跳过该镜头生成，提示"镜头 #X 的 Prompt 为空，已跳过" |
| 生成中途关闭页面/刷新 | 任务状态已持久化到数据库，重新进入页面时自动恢复任务状态 |
| 图片文件损坏/无法加载 | 图片区域显示错误图标，提供「重新生成」按钮 |
| 批量生成时部分任务失败 | 成功的保留结果，失败的标记错误，提供「重试所有失败项」 |
| 生成历史超过 3 条 | 自动删除最旧的历史图片文件，保留最新 3 条 |
| 点击确认时存在大量未生成镜头 | 弹窗额外提示"尚有 X 个镜头未生成画面，是否继续？" |

## 6.12 非功能性需求

| 维度 | 要求 |
|------|------|
| **并发控制** | 最多 4 并发任务 |
| **存储效率** | 图片以 PNG 格式保存，单张约 1-3MB |
| **UI 响应性** | 生成中 UI 不卡顿，进度更新实时反映到卡片状态 |
| **断点续传** | 进度状态持久化，关闭页面后重新进入可继续查看/恢复未完成任务 |
| **错误可见性** | 每个失败任务有清晰的错误原因展示 |
| **数据安全** | 图片文件保存在本地 `public/projects/` 目录，不自动上传云端 |
| **可替换性** | 所有 AI 生成结果均可手动上传图片替换 |
| **历史版本** | 每个镜头保留最多 3 个历史版本，可随时回滚 |
| **响应式布局** | 三栏布局在不同屏幕宽度下自适应 |
| **竖屏优化** | 默认输出 9:16 比例图像 |
