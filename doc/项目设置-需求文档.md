**文档版本**：1.0  
**更新日期**：2025-03-17  
**关联**：设计文档、模块1-项目管理

---

## 1. 模块定位

「项目设置」分为两层：

1. **全局项目设置**：在系统级别配置**新建项目时的默认参数**以及 **AI / 输出相关配置**，作用于所有项目。
2. **项目内设置**：在单个项目内配置**仅影响当前项目**的选项（如视觉风格），位于项目侧边栏「项目设置」分组下。

本需求文档同时覆盖上述两层，便于产品与开发统一理解与实现。

---

## 2. 全局项目设置（系统设置页）

**入口**：主导航或用户菜单 → 「设置」→ 页面标题「项目设置」。  
**描述**：管理全局默认配置，新建项目时自动应用。

### 2.1 默认项目配置

新建项目时，表单默认带出以下配置，用户可在创建时覆盖。

| 配置项 | 说明 | 可选值 | 必填 |
|--------|------|--------|------|
| **目标平台** | 画幅与分辨率预设 | 抖音 (9:16, 1080×1920)、TikTok (9:16, 1080×1920)、小红书 (3:4, 1080×1440)、YouTube (16:9, 1920×1080) | 是 |
| **默认时长** | 单集时长，影响剧本节奏 | 30 秒、60 秒、90 秒 | 是 |
| **默认视觉风格** | 新建项目的默认风格预设 | 写实-都市霸总、写实-古装仙侠、写实-硬核科幻、日漫风、美漫风格、赛博朋克、水墨风 | 是 |

- 支持保存到本地（如 localStorage 或服务端用户配置），下次进入设置页时回显。
- 保存后，新建项目表单自动带出上述默认值。

### 2.2 AI 模型配置

配置文本生成与图像生成所使用的 AI 服务，供各模块（剧本、分镜、画面等）调用。

| 配置项 | 说明 | 格式/示例 | 必填 |
|--------|------|-----------|------|
| **文本大模型 API Key** | 剧本生成、角色分析等文本任务 | 如 `sk-...`，密码输入框 | 使用前必填 |
| **图像模型 API Key** | 分镜画面生成、角色头像生成 | 如 `sk-...`，密码输入框 | 使用前必填 |
| **ComfyUI 服务地址** | 本地或远程 ComfyUI 实例 | URL，如 `http://127.0.0.1:8188` | 可选 |

- 说明文案：
  - 文本 Key：用于剧本生成、角色分析等文本处理任务。
  - 图像 Key：用于分镜画面生成、角色头像生成。
  - ComfyUI：本地或远程 ComfyUI 实例地址，用于高级图像生成。
- Key 需安全存储（环境变量或加密存储），前端仅展示脱敏或占位。

### 2.3 输出配置

视频导出与存储的全局默认值。

| 配置项 | 说明 | 可选值 | 必填 |
|--------|------|--------|------|
| **默认输出目录** | 导出视频的根目录 | 本地路径，如 `~/CutGo/output` | 否 |
| **默认分辨率** | 导出视频分辨率 | 720P、1080P、2K | 是 |
| **默认字幕** | 导出是否带字幕 | 带字幕、无字幕、同时导出两版 | 是 |

- 保存后，导出模块未单独配置时使用以上默认值。

### 2.4 保存与校验

- 提供「保存设置」按钮，提交后持久化（本地或服务端）。
- 可选：对 API Key、URL 做格式校验；路径存在性校验可提示但不阻断保存。
- 保存成功：Toast 或内联提示「设置已保存」。

## 3. 项目内设置（项目侧边栏）

**入口**：进入某一项目后，左侧边栏分组「项目设置」下的子项（如「视觉风格」）。  
**描述**：仅影响当前项目的配置，与全局默认值独立。

### 3.1 视觉风格管理器（核心）

**定位**：AI 绘画一致性的关键。在项目内锁定视觉参数，防止整剧画风漂移。

#### 3.1.1 风格预设 (Preset)

- **写实类**：都市霸总、古装仙侠、硬核科幻。
- **艺术类**：日漫风、美漫风格、赛博朋克、水墨风。
- 与全局「默认视觉风格」选项保持一致；项目内选择覆盖该项目使用的风格，不影响其他项目及全局默认。

#### 3.1.2 全局负面提示词 (Global Negative Prompts)

- 支持配置一段通用负面词（如 `bad anatomy, text, watermark, low quality`），在项目内生成画面时自动注入，减少常见瑕疵。

#### 3.1.3 风格参考图 (Style Reference)

- 支持上传一张「风格参考图」，系统通过 IP-Adapter 或类似能力提取色彩与笔触，确保全剧画面调性统一。
- 可选：支持清除、替换、预览。

#### 3.1.4 数据与存储

- 以上配置与当前项目 ID 绑定，写入项目元数据或独立「项目设置」表，随项目持久化。

### 3.2 扩展项（后续迭代）

- **项目级覆盖**：是否允许本项目覆盖全局的「默认时长」「目标平台」等（当前可与创建时一致，仅做展示或只读）。
- **资产库入口**：在「项目设置」下增加「角色库 / 场景库」入口，与模块 1 资产库规划一致，按开发计划实现。

## 4. 与其它模块的关系

| 模块 | 关系说明 |
|------|----------|
| **模块 1 - 项目管理** | 新建项目时读取「全局默认项目配置」；项目实体需支持存储「项目内设置」（如视觉风格、负面词、参考图）。 |
| **小说导入 / 分集大纲 / 剧本生成** | 不直接依赖项目设置；项目级风格在分镜与画面生成时使用。 |
| **分镜生成 / 画面生成** | 使用项目内「视觉风格」「负面词」「参考图」及全局「图像 API / ComfyUI」配置。 |
| **导出发布** | 使用全局「输出配置」默认值（输出目录、分辨率、字幕）。 |

---

## 5. 非功能需求

- **安全性**：API Key 不以明文展示、不写日志；优先使用服务端或本地安全存储。
- **可用性**：全局设置页结构清晰，分组明确（默认项目配置 / AI 模型配置 / 输出配置）；项目内设置入口在侧边栏固定可见。
- **一致性**：风格预设枚举与模块 1、创建项目表单、分镜/画面模块共用同一套选项与数据源。
