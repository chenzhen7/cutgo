# 📖 模块 2：小说导入

## 2.1 模块定位

作为生产线的"原料入口"，负责将用户的小说文本导入系统，通过 AI 智能分析，完成段落拆分、人物识别、冲突结构提取，输出"剧情结构预览"。这是用户体验的**第一个惊艳点**——用户粘贴一段文字，系统就能自动理解故事结构。

**前置依赖**：模块 1（项目管理）已完成，当前项目已创建。

**输出物**：结构化的小说数据（章节、段落、角色、事件、大纲），供模块 3（剧本生成）消费。

---

## 2.2 核心功能拆解

### 2.2.1 文本输入（Text Ingest）

用户将小说原文输入系统的入口，支持两种方式。

#### A. 粘贴文本

- 提供大文本输入区域（Textarea），支持粘贴长文本
- 实时显示字数统计（字符数 / 预估字数）
- 上限提示：单次导入建议不超过 50,000 字，超过时给出警告但不硬性阻止
- 支持清空重填

#### B. 上传 TXT 文件

- 支持 `.txt` 文件上传（UTF-8 编码）
- 拖拽上传 + 点击上传两种交互
- 文件大小限制：≤ 5MB
- 上传后自动将文件内容填入文本区域，用户可二次编辑
- 文件名自动提取，作为"原文标题"的默认值

#### C. 章节范围选择

- 如果文本中包含明显的章节标记（如"第X章"、"Chapter X"、"第X节"等），系统自动识别并展示章节列表
- 用户可勾选需要导入的章节范围（全选 / 部分选择）
- 章节识别规则：
  - 正则匹配常见格式：`第\d+章`、`第[一二三四五六七八九十百]+章`、`Chapter \d+`、`---` 分隔符等
  - 无法识别章节时，按段落自然分割，提示用户"未检测到章节结构，将按段落导入"

---

### 2.2.2 AI 智能分析（Novel Analysis）

点击"开始分析"后，系统调用 AI 大模型对导入文本进行结构化分析。

#### 分析流程

```
原始文本 → 段落拆分 → AI 分析请求 → 结构化结果 → 存储入库
```

#### 分析维度

| 维度 | 说明 | 输出 |
|------|------|------|
| **段落拆分** | 按章节/段落自然分割文本 | 段落列表（含序号、内容、字数） |
| **人物识别** | 从文本中提取所有出场人物 | 角色列表（名称、首次出场位置、出场频次、简要描述） |
| **冲突结构提取** | 识别故事的戏剧结构 | 事件列表（起因、发展、高潮、结局标注） |
| **故事大纲** | 生成一段精炼的故事梗概 | 纯文本摘要（200-500字） |
| **情感曲线** | 标注文本中的情感走向 | 情感标签序列（平静、紧张、悲伤、高潮、温馨等） |
| **关键场景** | 识别适合改编为视频的高价值片段 | 场景列表（位置、描述、推荐理由） |

#### AI Prompt 设计要点

- **角色**：你是一位专业的影视编剧顾问，擅长将小说文本转化为短剧结构
- **输出格式**：要求 AI 返回严格的 JSON 结构，便于前端解析
- **上下文窗口**：如文本过长（超过模型上下文限制），分段分析后合并结果
- **幂等性**：同一段文本多次分析结果应大致一致

#### 分析状态管理

| 状态 | 说明 | UI 表现 |
|------|------|---------|
| `idle` | 未开始 | 显示空状态提示 |
| `analyzing` | 分析中 | 进度指示器 + 阶段性文字提示（"正在拆分段落..."、"正在识别角色..."） |
| `completed` | 分析完成 | 展示完整结果面板 |
| `error` | 分析失败 | 错误提示 + 重试按钮 |

---

### 2.2.3 分析结果展示（Analysis Result Panel）

分析完成后，展示结构化的分析结果，分为多个 Tab 面板。

#### Tab 1：故事大纲

- 以卡片形式展示 AI 生成的故事梗概
- 显示核心冲突、主要角色、故事基调
- 支持用户手动编辑大纲文本（编辑后标记为"已修改"）

#### Tab 2：角色列表

以卡片网格展示识别出的角色：

| 字段 | 说明 |
|------|------|
| 角色名称 | AI 提取的人物名 |
| 角色类型 | 主角 / 配角 / 龙套（AI 根据出场频次和剧情重要性判断） |
| 首次出场 | 在原文中首次出现的位置（章节/段落） |
| 出场频次 | 在文本中被提及的次数 |
| 角色简介 | AI 生成的一句话描述（如："霸道总裁，外冷内热，家族企业继承人"） |
| 关系标签 | 与其他角色的关系（如：男主 ↔ 女主：恋人） |

- 支持手动添加 AI 未识别的角色
- 支持删除误识别的角色
- 支持编辑角色信息
- 角色数据将作为模块 4（角色生成）的输入源

#### Tab 3：剧情结构

以时间线/流程图形式展示故事的戏剧结构：

- **起因**（Setup）：故事背景建立
- **发展**（Rising Action）：冲突升级
- **高潮**（Climax）：核心对抗/转折
- **结局**（Resolution）：收束

每个节点包含：
- 事件摘要
- 对应原文段落引用
- 情感标签（紧张、温馨、悲伤、激昂等）
- 短剧改编价值评分（1-5星，AI 评估该片段的视觉冲击力和情感张力）

支持：
- 拖拽调整事件顺序
- 标记/取消标记"高潮点"
- 删除不需要的事件节点
- 手动添加事件节点

#### Tab 4：章节段落

展示导入文本的章节/段落结构：

- 树形结构：章节 → 段落
- 每个段落显示：序号、前30字预览、字数、关联角色标签
- 点击段落可展开查看完整内容
- 支持勾选"纳入剧本生成"的段落范围

---

### 2.2.4 导入确认与保存（Import Confirmation）

用户审阅分析结果后，确认导入。

#### 确认流程

1. 用户在分析结果中确认/调整角色列表、事件结构、章节选择
2. 点击"确认导入"按钮
3. 系统将以下数据持久化到数据库：
   - 原始文本
   - 章节段落结构
   - 角色列表
   - 事件/剧情结构
   - 故事大纲
4. 更新项目状态：`step = 2` → `step = 3`，`stepLabel = "剧本生成"`
5. 提示"导入成功"，引导用户进入下一步（剧本生成）

#### 重新分析

- 在"已完成分析"状态下，用户可修改原始文本并重新触发分析
- 重新分析会覆盖之前的分析结果（给出二次确认弹窗）
- 已确认导入的项目，仍可回到此页面查看/修改导入内容

## 2.3 数据模型设计

### Novel（小说原文）

```prisma
model Novel {
  id        String   @id @default(cuid())
  projectId String   @unique
  project   Project  @relation(fields: [projectId], references: [id], onDelete: Cascade)

  title     String?
  rawText   String
  wordCount Int
  source    String   @default("paste")  // "paste" | "file"
  fileName  String?

  synopsis  String?  // AI 生成的故事大纲

  status    String   @default("draft")   // "draft" | "analyzed" | "confirmed"

  chapters  Chapter[]
  characters NovelCharacter[]
  events    PlotEvent[]

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

### Chapter（章节）

```prisma
model Chapter {
  id        String   @id @default(cuid())
  novelId   String
  novel     Novel    @relation(fields: [novelId], references: [id], onDelete: Cascade)

  index     Int
  title     String?
  content   String
  wordCount Int
  selected  Boolean  @default(true)  // 是否纳入剧本生成

  paragraphs Paragraph[]

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

### Paragraph（段落）

```prisma
model Paragraph {
  id        String   @id @default(cuid())
  chapterId String
  chapter   Chapter  @relation(fields: [chapterId], references: [id], onDelete: Cascade)

  index     Int
  content   String
  wordCount Int
  selected  Boolean  @default(true)

  createdAt DateTime @default(now())
}
```

### NovelCharacter（小说角色——原始识别数据）

```prisma
model NovelCharacter {
  id        String   @id @default(cuid())
  novelId   String
  novel     Novel    @relation(fields: [novelId], references: [id], onDelete: Cascade)

  name         String
  role         String   @default("supporting")  // "protagonist" | "supporting" | "extra"
  description  String?
  firstAppear  String?   // 首次出场位置描述
  frequency    Int       @default(1)
  relations    String?   // JSON: [{ target: "角色B", relation: "恋人" }]

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

### PlotEvent（剧情事件）

```prisma
model PlotEvent {
  id        String   @id @default(cuid())
  novelId   String
  novel     Novel    @relation(fields: [novelId], references: [id], onDelete: Cascade)

  index       Int
  type        String    // "setup" | "rising" | "climax" | "resolution"
  summary     String
  detail      String?
  emotion     String?   // 情感标签
  sourceRef   String?   // 对应原文引用
  adaptScore  Int?      // 改编价值评分 1-5
  isHighlight Boolean   @default(false)  // 是否为高潮点

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

## 2.4 API 设计

### `POST /api/novels`

创建 / 导入小说原文。

**请求体：**

```json
{
  "projectId": "clxxx...",
  "title": "龙王归来",
  "rawText": "第一章 重生\n他缓缓睁开双眼...",
  "source": "paste",
  "fileName": null
}
```

**响应：** 返回创建的 Novel 对象（含 id）。

---

### `GET /api/novels?projectId=xxx`

获取指定项目的小说数据（含章节、角色、事件关联数据）。

---

### `PATCH /api/novels/:id`

更新小说信息（大纲、状态等）。

---

### `POST /api/novels/:id/analyze`

触发 AI 分析。

**流程：**

1. 从数据库读取 `rawText`
2. 调用 AI 大模型进行结构化分析
3. 将分析结果（章节、段落、角色、事件、大纲）批量写入数据库
4. 更新 Novel `status` 为 `analyzed`

**响应：**

```json
{
  "synopsis": "故事讲述了...",
  "chapters": [...],
  "characters": [...],
  "events": [...],
  "stats": {
    "totalWords": 12345,
    "chapterCount": 5,
    "characterCount": 8,
    "eventCount": 12
  }
}
```

**错误处理：**
- AI 服务不可用 → 返回 503，前端提示重试
- 文本为空 → 返回 400
- 文本过长 → 分段处理，返回合并结果

---

### `POST /api/novels/:id/confirm`

确认导入，更新小说状态为 `confirmed`，同时更新项目进度。

---

### `PUT /api/novels/:id/chapters/:chapterId`

更新章节信息（如选中状态）。

---

### `POST /api/novels/:id/characters`

手动添加角色。

### `PATCH /api/novels/:id/characters/:characterId`

编辑角色信息。

### `DELETE /api/novels/:id/characters/:characterId`

删除角色。

---

### `POST /api/novels/:id/events`

手动添加剧情事件。

### `PATCH /api/novels/:id/events/:eventId`

编辑事件。

### `DELETE /api/novels/:id/events/:eventId`

删除事件。

### `PUT /api/novels/:id/events/reorder`

拖拽调整事件顺序。

**请求体：**

```json
{
  "orderedIds": ["evt_1", "evt_3", "evt_2", "evt_4"]
}
```

## 2.5 状态管理（Zustand Store）

```typescript
interface NovelState {
  novel: Novel | null
  chapters: Chapter[]
  characters: NovelCharacter[]
  events: PlotEvent[]
  
  analysisStatus: "idle" | "analyzing" | "completed" | "error"
  analysisError: string | null

  // Actions
  importNovel: (projectId: string, data: ImportInput) => Promise<void>
  analyzeNovel: (novelId: string) => Promise<void>
  confirmImport: (novelId: string) => Promise<void>
  
  fetchNovel: (projectId: string) => Promise<void>
  updateSynopsis: (novelId: string, synopsis: string) => Promise<void>
  
  // 角色 CRUD
  addCharacter: (novelId: string, data: CharacterInput) => Promise<void>
  updateCharacter: (characterId: string, data: Partial<CharacterInput>) => Promise<void>
  deleteCharacter: (characterId: string) => Promise<void>
  
  // 事件 CRUD
  addEvent: (novelId: string, data: EventInput) => Promise<void>

  deleteEvent: (eventId: string) => Promise<void>
  reorderEvents: (novelId: string, orderedIds: string[]) => Promise<void>
  
  // 章节操作
  toggleChapterSelection: (chapterId: string) => Promise<void>
  toggleAllChapters: (selected: boolean) => Promise<void>
}
```

### 交互细节

1. **文本输入区**
   - 空态：显示占位文字"将小说内容粘贴到这里，或拖拽 txt 文件上传..."
   - 支持拖拽 txt 文件到文本区域，自动读取内容
   - 右下角实时字数统计
   - 超过 50,000 字时，字数显示变为警告色

2. **分析进度**
   - 点击"开始分析"后，按钮变为 loading 状态
   - 文本输入区变为只读
   - 在结果区域显示分析进度：
     - "正在拆分文本结构..." → "正在识别角色..." → "正在分析剧情结构..." → "正在生成大纲..."
   - 支持取消分析（中断 API 请求）

3. **结果面板**
   - 使用 Tabs 组件切换不同维度
   - 每个 Tab 右上角显示数据条目数（如"角色列表 (8)"）
   - 可编辑字段使用 inline-edit 模式（点击变为输入框）
   - 所有修改实时保存（debounce 300ms）

4. **确认导入**
   - 底部固定操作栏，包含"确认导入"主按钮
   - 点击后弹出确认弹窗，显示导入摘要：
     - 总字数、章节数、角色数、事件数
   - 确认后，页面自动跳转到剧本生成（`/project/[id]/script`）

## 2.7 边界情况与异常处理

| 场景 | 处理方式 |
|------|----------|
| 文本为空点击分析 | 按钮禁用（disabled） |
| 文本过短（< 100 字） | 提示"文本内容过短，建议至少导入 100 字以上的小说内容" |
| 上传非 txt 文件 | 提示"仅支持 .txt 格式文件" |
| 上传文件编码非 UTF-8 | 尝试自动检测编码并转换，失败则提示用户转为 UTF-8 |
| AI 分析超时（> 60s） | 显示超时提示 + 重试按钮 |
| AI 分析返回格式异常 | 降级处理：仅展示能解析的部分，其余显示"分析失败" |
| 网络断开 | 提示网络错误，保留已输入的文本 |
| 重复导入（已有小说数据） | 弹窗提示"当前项目已有导入数据，重新导入将覆盖原有内容"，需用户二次确认 |
| 无法识别章节结构 | 按自然段落分割，提示用户 |
| 角色识别为 0 个 | 提示"未检测到角色，请手动添加" |

## 2.8 与其他模块的数据流

```
模块1(项目管理)                模块2(小说导入)               模块3(剧本生成)
     │                              │                           │
     │  projectId                   │                           │
     ├─────────────────────────────►│                           │
     │                              │                           │
     │                              │  Novel                    │
     │                              │  ├─ synopsis ────────────►│ 故事大纲
     │                              │  ├─ chapters[] ──────────►│ 原文素材
     │                              │  ├─ characters[] ────────►│ 角色基础数据
     │                              │  └─ events[] ────────────►│ 剧情结构
     │                              │                           │
     │                              │         模块4(角色生成)     │
     │                              │              │            │
     │                              │  characters[]│            │
     │                              ├─────────────►│            │
     │                              │              │            │
```

- **向上游**：从 Project 获取 `projectId` 和项目配置（平台、时长等）
- **向下游**：
  - 模块 3（剧本生成）：消费故事大纲、章节内容、角色列表、事件结构
  - 模块 4（角色生成）：消费 `NovelCharacter` 作为角色创建的初始数据

## 2.9 非功能性需求

| 维度 | 要求 |
|------|------|
| **性能** | 50,000 字文本的本地处理（章节拆分、字数统计）应在 500ms 内完成 |
| **AI 响应** | 单次分析控制在 30s 内（取决于模型和文本长度），超时需有降级策略 |
| **数据安全** | 文本数据仅存本地 SQLite，不上传到云端（AI 调用除外） |
| **可用性** | 分析中断后已输入的文本不丢失（存入 localStorage 作为备份） |
| **可编辑性** | 所有 AI 分析结果均可手动修改，AI 只是辅助而非决定 |
