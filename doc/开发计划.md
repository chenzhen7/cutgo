根据需求文档和设计文档，项目共有 **8 个一级模块**，遵循用户流程 Step 1 → Step 8 的管线式架构。当前项目已搭建好基本的路由骨架、布局系统和 UI 组件库，但**业务逻辑、数据库、状态管理、API 层均未实现**。

***

### 基础设施层（地基）

> 所有模块都依赖这一层，必须首先完成。

**&#x20;API 路由层搭建**

* 在 `app/api/` 下创建 RESTful API 路由：

  * `app/api/projects/` — CRUD

  * `app/api/novels/` — 导入与解析

  * `app/api/scripts/` — 剧本生成与编辑

  * `app/api/characters/` — 角色管理

  * `app/api/storyboards/` — 分镜管理

  * `app/api/images/` — 图片生成

  * `app/api/videos/` — 视频合成

  * `app/api/exports/` — 导出

***

### 开发优先级总结

| 优先级 | 阶段  | 模块                       | 核心原因                |
| --- | --- | ------------------------ | ------------------- |
| P0  | 阶段一 | 基础设施（DB/Store/API/Bug修复） | 所有功能的前置依赖           |
| P0  | 阶段二 | 项目管理                     | 产品入口，"地基"模块         |
| P1  | 阶段三 | 小说导入                     | 数据源头，第一个惊艳点         |
| P1  | 阶段四 | 剧本生成                     | 核心竞争力，技术含量最高        |
| P2  | 阶段五 | 角色生成                     | 视觉一致性关键，可先简单实现      |
| P2  | 阶段六 | 分镜生成                     | 专业感来源，依赖剧本数据        |
| P3  | 阶段七 | 画面生成                     | 依赖角色+分镜，需 AI 图像 API |
| P3  | 阶段八 | 视频合成                     | 依赖画面，需 FFmpeg 集成    |
| P4  | 阶段九 | 导出发布                     | 最终环节，相对简单           |
