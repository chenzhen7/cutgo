根据需求文档和设计文档，项目共有 **9 个一级模块**，遵循用户流程 Step 1 → Step 9 的管线式架构。

***

### 基础设施层（地基）

> 所有模块都依赖这一层，必须首先完成。

** API 路由层搭建**

* 在 `app/api/` 下创建 RESTful API 路由：

  * `app/api/projects/` — CRUD ✅

  * `app/api/novels/` — 导入与解析 ✅

  * `app/api/episodes/` — 分集数据（确认导入时由章节生成）

  * `app/api/scripts/` — 剧本生成与编辑

  * `app/api/characters/` — 角色管理

  * `app/api/script-shots/` — 分镜管理

  * `app/api/images/` — 图片生成

  * `app/api/videos/` — 视频合成

  * `app/api/exports/` — 导出

### 管线步骤定义（pipeline.ts）

```
Step 1: 小说导入 → Step 2: 剧本生成 → Step 3: 角色生成
→ Step 4: 分镜生成 → Step 5: 画面生成 → Step 6: 视频合成 → Step 7: 导出发布
（确认导入时按章节自动生成分集，无独立「分集大纲」步骤。）
```
