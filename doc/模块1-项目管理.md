# 📂 模块 1：项目管理

## 1.1 模块定位

作为生产线的“中控室”，负责定义项目的全局配置（如画幅、风格、语种），并管理所有原始素材（小说原文）与生成资产（角色卡、场景库）的关联关系。

---

## 1.2 核心功能拆解

### **项目列表页（首页）**

* 实现项目的列表/网格视图切换

* 展示：项目名称、封面、总集数/时长、最后修改时间、状态标签

* 支持搜索、筛选、排序

* 对接数据库实际数据（替代现有 MOCK 数据）



### 1.2.1 项目创建与元数据设置 (Project Ingest)

用户发起新创作任务时的初始化配置。

- **项目属性配置**：
  - **项目名称**：自定义（如：*《龙王回归》第1-10集*）。
  - **描述**
  - **标签**
  - **封面：默认第一帧**
  - **目标平台预设**：
    - **抖音/TikTok** (9:16, 1080x1920)
    - **视频号/快手** (3:4, 1080x1440)
    - **YouTube/横屏** (16:9, 1920x1080)
  - **预计时长**：单集（30s / 60s / 90s / 自定义），影响 AI 拆解剧本的节奏感。

* 表单校验与提交

* 创建成功后跳转到项目工作流

### 1.2.2 视觉风格管理器 (Visual Style Manager)

**这是 AI 绘画一致性的关键。** 在项目开始前锁定视觉参数，防止视频风格“打架”。

- **风格预设 (Preset)**：
  - **写实类**：都市霸总、古装仙侠、硬核科幻。
  - **艺术类**：日漫风、美漫风格、赛博朋克、水墨风。
- **全局提示词 (Global Negative Prompts)**：
  - 自动注入通用负面词（如：`bad anatomy, text, watermark, low quality`）。
- **参考图上传 (Style Reference)**：
  - 支持上传一张“风格参考图”，系统通过 IP-Adapter 提取色彩和笔触，确保全剧画面调性统一。

### 1.2.3 资产库 (暂时不实现功能，只提供原型)

项目内所有生成内容的集中仓库，支持跨章节调用。

- **角色库 (Cast)**：展示本项目已定义的角色模型（Seed 值、人脸特征、固定服装）。
- **场景库 (Scene Assets)**：存储已生成的固定背景（如：*总裁办公室、荒野废墟*），确保场景在不同镜头间不穿帮。
- **道具库**
- **分镜资源**
- **素材复用逻辑**：支持将 A 项目的角色/场景直接导入 B 项目（系列剧必备）。

资产库暂时不实现功能，只提供原型，资产库和后面的模块关系比较深

### 1.2.4 项目看板

- **列表/网格视图**：展示所有创作中的短剧项目。
- **关键信息外显**：项目名称、封面图（自动抓取第一镜）、总集数/总时长、最后修改时间。
